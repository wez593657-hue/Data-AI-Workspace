# 01 AI SOP（工作流程规范）

## 1.1 概述

本规范定义了 AI 在 CRM 数据开发过程中的标准工作流程，确保所有 AI 工具（ChatGPT / Cursor / Claude Code / GitHub Copilot）输出一致、可维护、可追溯的代码。

## 1.2 AI 工作流

所有 AI 工具必须严格按照以下 9 个步骤执行：

```
步骤 1: 理解业务需求
        │
        ▼
步骤 2: 检查缺失信息
        │
        ▼
步骤 3: 输出设计方案
        │
        ▼
步骤 4: 编写 SQL / ETL / Procedure
        │
        ▼
步骤 5: Explain Plan 分析
        │
        ▼
步骤 6: 性能优化
        │
        ▼
步骤 7: 风险分析
        │
        ▼
步骤 8: Code Review
        │
        ▼
步骤 9: 输出最终结果
```

### 步骤 1：理解业务需求

- 阅读需求文档
- 提取核心业务逻辑
- 识别关键数据实体
- 确认业务规则和约束

**输出：**
```
业务理解：
- 需求背景：[描述]
- 核心目标：[描述]
- 业务规则：[描述]
- 数据实体：[列表]
```

### 步骤 2：检查缺失信息

**必须检查以下内容：**

| 检查项 | 检查内容 | 缺失标记 |
|--------|----------|----------|
| 数据字典 | 相关表和字段是否已定义 | 【待确认】 |
| Mapping | 字段映射是否已定义 | 【待确认】 |
| 表结构 | 表名、字段名、数据类型 | 【待确认】 |
| 业务规则 | 业务逻辑是否清晰 | 【待确认】 |
| 上游数据 | 数据来源是否明确 | 【待确认】 |
| 下游影响 | 对其他模块的影响 | 【待确认】 |

**要求：**
- 任何不确定内容必须使用【待确认】标记
- 禁止猜测或假设未确认的信息
- 必须列出所有缺失信息清单

### 步骤 3：输出设计方案

**设计方案必须包含：**

1. **数据模型设计**
   - 涉及的表结构
   - 字段定义
   - 主键/外键关系

2. **处理流程设计**
   - 数据流向图
   - 处理步骤
   - 转换逻辑

3. **技术选型**
   - SQL / 存储过程 / ETL
   - 临时表策略
   - 事务处理方式

4. **性能考虑**
   - 数据量评估
   - 索引策略
   - 执行计划预估

### 步骤 4：编写 SQL / ETL / Procedure

**编码规范：**
- 严格遵循 SQL 规范（02_SQL_Standard.md）
- 严格遵循存储过程规范（05_Stored_Procedure.md）
- 严格遵循 ETL 规范（06_ETL_Standard.md）
- 字段命名统一
- 添加必要注释

### 步骤 5：Explain Plan 分析

**必须输出：**
- 执行计划
- 索引使用情况
- 扫描类型（全表扫描/索引扫描）
- 预估执行时间
- 预估数据量

**分析要点：**
- 是否存在全表扫描
- 索引是否有效使用
- Join 顺序是否合理
- 是否存在临时表溢出

### 步骤 6：性能优化

**优化策略：**
1. 添加缺失索引
2. 优化 Join 顺序
3. 拆分复杂查询
4. 使用临时表
5. 限制返回数据量
6. 避免不必要的排序

**优化后必须重新执行 Explain Plan**

### 步骤 7：风险分析

**风险评估维度：**

| 风险类型 | 评估内容 | 严重程度 |
|----------|----------|----------|
| 数据一致性 | 是否影响数据准确性 | 高/中/低 |
| 性能风险 | 是否存在性能瓶颈 | 高/中/低 |
| 安全风险 | 是否存在 SQL 注入等安全问题 | 高/中/低 |
| 兼容性 | 是否兼容现有系统 | 高/中/低 |
| 回滚风险 | 是否支持回滚 | 高/中/低 |

**风险应对措施：**
- 高风险：必须提供解决方案
- 中风险：建议提供缓解措施
- 低风险：记录并监控

### 步骤 8：Code Review

**Review 内容：**
- SQL 正确性
- 性能
- Explain Plan
- 索引
- 命名
- 注释
- Mapping
- 数据字典
- 存储过程
- ETL
- 安全
- 风险

**Review 未通过，禁止进入下一步**

### 步骤 9：输出最终结果

**输出格式必须包含：**

```
业务理解
实现方案
代码
执行逻辑
性能分析
风险分析
优化建议
Code Review
```

## 1.3 AI 输出统一模板

所有 AI 输出必须遵循以下统一格式：

```text
=======================
【业务理解】
=======================
[描述业务需求、核心目标、业务规则]

=======================
【实现方案】
=======================
[描述技术方案、数据模型、处理流程]

=======================
【代码】
=======================
[SQL / ETL / 存储过程代码]

=======================
【执行逻辑】
=======================
[描述代码执行步骤、数据流向]

=======================
【性能分析】
=======================
[Explain Plan 结果、性能评估]

=======================
【风险分析】
=======================
[风险评估、应对措施]

=======================
【优化建议】
=======================
[性能优化建议、代码改进建议]

=======================
【Code Review】
=======================
[Review 结果、问题记录]
```

## 1.4 禁止事项

- ❌ 禁止跳过任一关键步骤
- ❌ 禁止直接编写 SQL 而不理解业务
- ❌ 禁止猜测未确认的信息
- ❌ 禁止自由发挥导致风格不一致
- ❌ 禁止省略关键输出部分
