# ODS 与 TMP 治理加固设计

## 目标

修复 ODS 数据字典覆盖人工内容、ODS/TMP 分层混淆、TMP_ 字段审核未验证、PR 审批不强制和本地钩子交互阻塞的问题。

## ODS 数据字典

生成器默认采用保守更新：已有数据字典中的人工字段（中文名称、字段中文说明、枚举说明、负责人）必须保留；仅以 DDL 更新表名、字段名、类型、长度、可空性、默认值和约束。新增 DDL 才创建含 `【待确认】` 的字典骨架。

`data_assets/ddl/ods/temp/` 不属于 ODS 原始数据资产，生成器与 ODS 一致性校验均排除该目录。

ODS 与 DWD、DWS、ADS 一样纳入字段级一致性校验。

## TMP 审核

TMP_ 校验器解析存储过程中的建表 DDL，按表名比对审核清单的字段名、类型、长度和可空性；清单还必须包含索引、清理方式和并发隔离策略。受保护分支 PR 要求 `approval_status` 为 `approved`。

## 门禁

PR 审批检查在无批准 Review 时返回失败。本地文档审核不再读取交互输入，发现规则冲突时直接失败并要求开发者修复或通过 PR 审核处理。

GitHub 仓库设置仍必须启用保护分支、Code Owner 审批和必需状态检查；工作流负责报告，平台设置负责禁止合并。

## 验证

验证既有 ODS 字典人工字段不被重写、ODS 结构漂移被识别、TMP_ 字段不一致被拒绝、未批准 PR 审核失败，以及文档校验不会阻塞等待输入。
