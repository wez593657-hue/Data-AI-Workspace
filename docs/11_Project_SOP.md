# 11 项目 SOP

## 11.1 概述

本规范定义了 CRM 数据开发项目的标准流程，确保项目按计划有序推进，禁止跳过任一关键步骤。

## 11.2 项目开发流程

```
需求分析
    │
    ▼
需求确认
    │
    ▼
数据分析
    │
    ▼
数据字典确认
    │
    ▼
Mapping设计
    │
    ▼
SQL设计
    │
    ▼
存储过程设计
    │
    ▼
ETL设计
    │
    ▼
Explain分析
    │
    ▼
Code Review
    │
    ▼
测试
    │
    ▼
上线
```

## 11.3 流程详细说明

### 11.3.1 需求分析

**输入：**
- 业务需求文档
- 相关系统文档
- 历史数据

**输出：**
- 需求分析报告
- 业务规则说明
- 数据实体清单

**执行要点：**
- 深入理解业务背景和目标
- 识别关键数据实体和业务规则
- 确定数据来源和流向
- 记录不确定事项（【待确认】）

### 11.3.2 需求确认

**输入：**
- 需求分析报告

**输出：**
- 需求确认书（签字/审批）
- 修订后的需求文档

**执行要点：**
- 与业务方确认需求理解正确
- 解决【待确认】事项
- 获取业务方签字确认
- 冻结需求范围

### 11.3.3 数据分析

**输入：**
- 需求确认书
- 源系统数据

**输出：**
- 数据分析报告
- 数据质量评估
- 数据量预估

**执行要点：**
- 分析源系统数据结构
- 评估数据质量（完整性、准确性、一致性）
- 预估数据量和增长趋势
- 识别数据问题和风险

### 11.3.4 数据字典确认

**输入：**
- 数据分析报告
- 需求确认书

**输出：**
- 数据字典文档
- 表结构设计
- 字段定义

**执行要点：**
- 按照数据字典规范编写（07_Data_Dictionary.md）
- 确认表名、字段名、数据类型
- 定义主键、外键、约束
- 与业务方确认数据字典

### 11.3.5 Mapping 设计

**输入：**
- 数据字典
- 源系统数据结构

**输出：**
- Mapping 文档
- 转换规则说明

**执行要点：**
- 按照 Mapping 规范编写（08_Mapping.md）
- 定义每个字段的来源和转换规则
- 确保 Mapping 与数据字典一致
- 标注数据追溯路径

### 11.3.6 SQL 设计

**输入：**
- 数据字典
- Mapping 文档

**输出：**
- SQL 代码
- 执行逻辑说明

**执行要点：**
- 按照 SQL 规范编写（02_SQL_Standard.md）
- 确保 SQL 与数据字典、Mapping 一致
- 添加必要注释
- 禁止 SELECT *

### 11.3.7 存储过程设计

**输入：**
- SQL 代码
- 业务规则

**输出：**
- 存储过程代码
- 接口说明
- 临时表建表语句（`data_assets/ddl/tmp/tmp_存储过程名.ddl`）

**执行要点：**
- 按照存储过程规范编写（05_Stored_Procedure.md）
- 包含参数检查、事务处理、异常捕获、日志记录
- 定义统一返回码
- 支持回滚
- **强制要求**：同步生成/更新临时表建表语句文件，包含存储过程中所有 `TMP_` 前缀的物理中间表

### 11.3.8 ETL 设计

**输入：**
- Mapping 文档
- SQL 代码
- 数据字典

**输出：**
- ETL 脚本
- ETL 配置
- 调度计划

**执行要点：**
- 按照 ETL 规范编写（06_ETL_Standard.md）
- 支持幂等性和可重跑
- 包含数据校验和日志记录
- 设计合理的调度策略

### 11.3.9 Explain 分析

**输入：**
- SQL 代码
- 存储过程代码

**输出：**
- Explain Plan 结果
- 性能分析报告
- 优化建议

**执行要点：**
- 执行 Explain Plan 分析（03_SQL_Performance.md）
- 识别性能瓶颈
- 提出优化方案
- 验证优化效果

### 11.3.10 Code Review

**输入：**
- SQL 代码
- 存储过程代码
- ETL 脚本
- 数据字典
- Mapping 文档

**输出：**
- Review 报告
- 问题修复记录

**执行要点：**
- 按照 Code Review 规范执行（10_Code_Review.md）
- 检查 SQL 正确性、性能、安全性
- 检查 Mapping 和数据字典一致性
- Review 未通过，禁止进入下一步

### 11.3.11 测试

**输入：**
- 所有代码和文档

**输出：**
- 测试报告
- Bug 修复记录

**测试类型：**

| 测试类型 | 测试内容 | 负责人 |
|----------|----------|--------|
| 单元测试 | SQL 语句正确性 | 开发人员 |
| 集成测试 | 模块间接口正确性 | 开发人员 |
| 性能测试 | 执行时间、吞吐量 | 测试人员 |
| 数据验证 | 数据一致性、准确性 | 测试人员 |
| 回归测试 | 不影响现有功能 | 测试人员 |

### 11.3.12 上线

**输入：**
- 测试报告（通过）
- Review 报告（通过）
- 所有代码和文档

**输出：**
- 上线确认书
- 运行监控报告

**执行要点：**
- 执行上线前检查清单
- 备份数据
- 灰度发布（可选）
- 监控运行状态
- 记录上线日志

- ❌ 禁止在数据字典未确认时编写 SQL
