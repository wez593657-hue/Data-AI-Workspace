# 10 Code Review 规范

## 10.1 概述

Code Review 是确保代码质量的关键环节，必须在代码上线前完成。Review 未通过，禁止上线。

## 10.2 Review 流程

```
提交代码 → 分配 Reviewer → 执行 Review → 提出问题 → 修复问题 → 重新 Review → 确认通过
```

## 10.4 审查标准

| 标准 | 要求 |
|------|------|
| SQL 正确性 | 字段名、表名与数据字典一致，JOIN 条件正确，WHERE 条件合理 |
| 性能 | 有效使用索引，避免全表扫描，执行计划合理 |
| 代码规范 | 命名符合规范，添加必要注释，格式整洁 |
| 数据一致性 | Mapping、数据字典、DDL 保持一致 |
| 安全性 | 无 SQL 注入风险，敏感数据脱敏处理 |
| 风险评估 | 识别潜在风险并提供应对措施 |

## 10.5 Review 等级

| 等级 | 定义 | 处理方式 |
|------|------|----------|
| P0 - 阻塞 | 严重错误，可能导致数据错误或系统崩溃 | 必须修复后才能通过 |
| P1 - 高 | 重要问题，影响功能正确性或性能 | 建议修复后再通过 |
| P2 - 中 | 一般问题，建议改进 | 可在后续版本修复 |
| P3 - 低 | 轻微问题，不影响功能 | 记录并在后续优化 |

## 10.7 Review 检查清单

| 检查项 | 检查内容 | 状态 |
|--------|----------|------|

| 索引 | 必要的索引是否存在 | ✅/❌ |
| 命名 | 是否符合命名规范 | ✅/❌ |
| 注释 | 是否添加必要注释 | ✅/❌ |
| Mapping | 是否与 Mapping 一致 | ✅/❌ |
| 数据字典 | 是否与数据字典一致 | ✅/❌ |

| 风险 | 是否评估风险并提供应对措施 | ✅/❌ |

## 10.8 禁止事项

- ❌ 禁止跳过 Code Review 直接上线
- ❌ 禁止忽略 P0 和 P1 级问题
- ❌ 禁止在 Review 未通过时强行上线
- ❌ 禁止不记录 Review 问题
- ❌ 禁止不修复已发现的问题
