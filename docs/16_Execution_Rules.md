# 16 执行规则

## 16.1 核心原则

所有操作必须严格遵循"用户命令驱动"原则，禁止擅自执行任何变更。

### 16.1.1 禁止自动执行的操作

| 操作类型 | 禁止行为 | 正确做法 |
|----------|----------|----------|
| 代码生成 | 自动生成存储过程、ETL脚本、DDL等 | 等待用户明确命令 |
| 文件修改 | 擅自修改数据资产（DDL、字典、Mapping） | 等待用户明确命令 |
| 数据清理 | 自动删除文件或目录 | 等待用户明确命令 |
| 提交推送 | 自动执行git commit/push | 等待用户明确命令 |
| 校验修复 | 自动执行ai_repair_loop修复 | 等待用户明确命令 |

### 16.1.2 允许自动执行的操作

| 操作类型 | 允许行为 | 说明 |
|----------|----------|------|
| 校验检查 | 执行workspace_validation.py | 仅检查，不修改 |
| 状态查询 | git status, git log | 只读操作 |
| 项目分析 | 查看目录结构、文件内容 | 只读操作 |
| 规则检查 | 检查是否符合规范 | 仅检查，不修改 |

## 16.2 执行流程

### 16.2.1 修改流程

```
用户命令 → 分析需求 → 确认方案 → 执行操作 → 验证结果 → 等待确认
     │            │           │           │           │
     ▼            ▼           ▼           ▼           ▼
  明确意图     理解范围    用户确认    执行修改    验证输出
```

### 16.2.2 数据开发流程

```
用户提供需求文档
        │
        ▼
用户命令生成DDL
        │
        ▼
用户命令生成数据字典
        │
        ▼
用户命令生成Mapping
        │
        ▼
用户命令生成存储过程
```

## 16.3 操作规范

### 16.3.1 修改前确认

所有修改操作前必须：
1. 明确告知用户即将执行的操作内容
2. 说明影响范围和潜在风险
3. 等待用户确认后再执行

### 16.3.2 修改后验证

所有修改操作后必须：
1. 执行相应的校验脚本验证修改正确性
2. 提供修改结果摘要
3. 等待用户确认后再提交

### 16.3.3 提交规范

提交前必须：
1. 执行完整工作区校验（workspace_validation.py full）
2. 确认所有校验通过
3. 等待用户明确命令后执行git commit/push

## 16.4 权限边界

| 权限级别 | 允许操作 | 禁止操作 |
|----------|----------|----------|
| 只读权限 | 查看、分析、校验 | 修改、删除、生成 |
| 修改权限 | 按用户命令修改指定文件 | 修改未授权文件 |
| 提交权限 | 按用户命令提交 | 擅自提交 |

## 16.5 违规处理

| 违规类型 | 处理方式 |
|----------|----------|
| 擅自生成代码 | 立即停止，删除生成内容，向用户道歉 |
| 擅自修改数据资产 | 立即停止，恢复原始状态，向用户道歉 |
| 擅自提交推送 | 立即撤销，恢复原始状态，向用户道歉 |

## 16.6 命令清单

### 16.6.1 常用命令

| 命令 | 说明 |
|------|------|
| `校验` | 执行workspace_validation.py full |
| `快速校验` | 执行workspace_validation.py quick |
| `修复` | 执行ai_repair_loop.py |
| `提交` | git commit |
| `推送` | git push |
| `提交并推送` | git commit && git push |

### 16.6.2 数据开发命令

| 命令 | 说明 |
|------|------|
| `生成DDL` | 根据需求文档生成DDL脚本 |
| `生成数据字典` | 根据DDL生成数据字典 |
| `生成Mapping` | 根据DDL和数据字典生成Mapping |
| `生成存储过程` | 根据DDL、数据字典、Mapping生成存储过程 |

### 16.6.3 管理命令

| 命令 | 说明 |
|------|------|
| `查看状态` | git status |
| `查看日志` | git log |
| `分析项目` | 分析项目结构和状态 |
