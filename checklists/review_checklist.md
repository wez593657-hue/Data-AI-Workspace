# 审查检查清单

## 一、代码规范性审查

### 1.1 SQL规范

- [ ] 表名符合命名规范
- [ ] 字段命名统一
- [ ] SQL语句格式清晰
- [ ] 注释完整准确

### 1.2 数据字典规范

- [ ] 数据字典结构完整
- [ ] 字段描述准确
- [ ] 数据类型一致
- [ ] 业务含义清晰

### 1.3 Mapping规范

- [ ] 映射关系完整
- [ ] 转换规则明确
- [ ] 来源表名正确
- [ ] 字段名对应准确

## 二、逻辑正确性审查

### 2.1 业务逻辑

- [ ] 实现符合需求文档
- [ ] 边界条件处理正确

- [ ] 数据流转正确

### 2.2 数据一致性

- [ ] DDL与数据字典一致

- [ ] 上下游数据字段一致
- [ ] 主键外键关系正确

### 2.3 ETL逻辑

- [ ] 抽取逻辑正确
- [ ] 转换规则正确
- [ ] 加载逻辑正确
- [ ] 增量更新策略合理

### 3.1 数据安全

- [ ] 敏感字段已脱敏
