我的申请

我的申请-列表

功能描述

支持用户集中查看本人发起的各类申请记录，按审批状态分类跟踪申请结果和流程进度，提升申请查询与跟进效率。

输入

“审批中”“审批通过”“审批未通过”三个Tab页面，查询条件区的查询条件一致，即申请类型、流程状态、申请时间。

处理

（一）操作说明

1. 用户进入“我的申请”列表页面后，页面按“申请状态”分为“审批中”“审批通过”“审批未通过”三个Tab页面。系统默认展示“审批中”Tab列表。

2. 查询条件区的操作按钮有：“查询”“重置”。点击“查询”，系统按当前输入条件返回符合条件的申请列表结果；点击“重置”，系统清空当前输入条件并恢复初始状态。“审批中”“审批通过”“审批未通过”列表页的数据分开查询。

3. 列表左上角统一操作按钮区无按钮。

4. 列表行内统一操作按钮区的操作按钮有：“查看”。点击“查看”按钮，系统从右侧抽屉式打开“申请详情”页面，展示“申请内容”与“审批流程”信息。

（二）规则与约束

1. “审批中”列表展示申请提交后审批流程尚未结束的数据；“审批通过”列表展示审批流程已结束且全部节点均为“同意”的数据；“审批未通过”列表展示审批流程中任一审批节点为“未通过”的数据。

2. 申请类型有：“客户移交”“客户申领”“目标任务”“沙龙活动”“短信模板”。

3. “流程状态”取值为“正常”“异常”。系统处理审批流程中的数据错误并标记为异常的，流程状态显示“异常”，其他情况均显示“正常”。

4. 列表默认按“申请时间”倒序排序。

（三）权限说明

1. 菜单权限为业务岗、网点负责人、客户经理岗可见。

2. 数据权限为：申请者仅可查看本人提交的申请数据；审批者（即在审批节点上的用户）可查看所有申请者提交的申请数据。

输出

“审批中”tab页：

“审批通过”tab页：

“审批未通过”tab页：

我的申请-详情

功能描述

支持用户查看单笔申请的详情，即审批信息和审批流程，准确掌握申请内容（即审批信息）、审批节点状态及处理进度。

输入

不涉及。

处理

（一）操作说明

1. 用户在“我的申请”列表行内统一操作按钮区点击“查看”，系统从右侧抽屉式打开“申请详情”页面。

2. 页面上部分展示“审批信息”，下部分展示“审批流程”。

2.1 “审批信息”：不同的申请类型，展示的审批信息不同，详见此模块的“输出”章节。

2.2 “审批流程”：按顺序展示申请人、审批节点。申请人展示申请人、申请时间。审批节点按系统配置的审批顺序展示审批节点，未完成审批的节点只展示审批人，且节点图标为灰色，已完成审批的节点展示审批人、审批时间、审批意见，且节点图标为亮色。

3. 用户点击右下角操作按钮区“确定”按钮后，系统关闭“申请详情”页面。

（二）规则与约束

1. 页面上部分“审批信息”根据申请类型展示不同明细内容；下部分“审批流程”按审批顺序展示申请人节点及审批节点信息。

2. 审批流程中，申请人节点展示申请人、申请时间；审批节点按系统配置顺序展示。未完成审批的节点仅展示审批人字段，节点图标为灰色；已完成审批的节点展示审批人、审批时间、审批意见，节点图标为亮色。

3. 沙龙活动参与人中的手机号码属于敏感信息，需脱敏展示，可按“保留前3位和后4位，中间4位以*替代，如138****1234”的规则显示。

（三）权限说明

1. 申请者可查看本人提交申请的详情。

2. 位于审批节点上的审批者可查看该申请的详情。

待办审批

待办审批-列表

功能描述

支持用户集中查看本人待审批及已审批的申请事项，并按审批状态分类执行审批处理或查看审批结果。

输入

“审批中”“已审批”两个Tab页面，查询条件区的查询条件一致，即申请类型、申请时间、申请人、流程状态进行查询。

处理

（一）操作说明

1. 用户进入“待办审批”列表页面后，页面按“审批状态”分为“审批中”“已审批”两个Tab页面。系统默认展示“审批中”Tab页面。

2. 查询条件区的操作按钮有：“查询”“重置”。点击“查询”，系统按当前输入条件返回符合条件的列表结果；点击“重置”，系统清空当前输入条件并将列表恢复初始状态。“审批中”、“已审批”列表页的数据分开查询。

3. “审批中”列表行内统一操作按钮区的操作按钮有：审批。点击“审批”按钮，系统从右侧抽屉式打开待审批“审批详情”页面。

4. “已审批”列表行内统一操作按钮区的操作按钮有：查看。点击“查看”按钮，系统从右侧抽屉式打开已审批“审批详情”页面。

5. 列表左上角统一操作按钮区无按钮。

（二）规则与约束

1. “审批中”展示“待当前节点审批”且“用户位于当前审批节点中”的申请数据；“已审批”展示“用户位于审批节点中”且“该审批节点已审批”的申请数据。审批节点中的用户共享审批信息，即节点中的任一用户完成审批后，节点中的其他用户也可查看该节点的“审批意见”等数据。

2. 审批中列表按“申请时间”倒序排序；已审批列表按“审批时间”倒序排序。

3. 流程状态取值为“正常”“异常”。系统处理审批流程中的数据错误并标记为异常的，流程状态显示“异常”，其他情况均显示“正常”。

（三）权限说明

1. 菜单权限为总分支行业务岗、网点负责人。

2. 数据权限为：审批者（即在审批节点上的用户）可查看流转到该节点的所有申请数据。

3. 审批者可查看并处理待办审批列表。

“已审批”tab页：

支持审批者在审批详情中查看申请明细和审批流程，录入审批意见并执行同意或拒绝操作，完成当前审批节点处理。

输入

处理

（一）操作说明

1. 用户在“审批中”列表行内统一操作按钮区点击“审批”按钮，系统从右侧抽屉式打开待审批的“审批详情”页面。

2. 用户在“已审批”列表行内统一操作按钮区点击“查看”按钮，系统从右侧抽屉式打开已审批的“审批详情”页面。

3. “审批详情”页面分为“审批信息”“审批意见”“审批流程”三个区域。待审批的“审批详情”页面，“审批意见”可编辑；已审批的“审批详情”页面，“审批意见”不可编辑，仅做数据回显。

3. “审批详情”页面下部分“审批流程”的展示内容，与“我的申请”详情页下部分的“审批流程”保持一致。详见“待办及消息提醒-我的申请-详情”章节。

（三）权限说明

1. 仅位于当前审批节点上的审批者可在待审批详情页执行“同意”或“拒绝”操作。

2. 已审批详情页仅用于查看，审批意见不可修改，不可重复审批。

大额动账

功能描述

支持用户集中查看管辖客户的大额资金变动提醒，及时识别转入、转出等异常或重点资金流动并跟进客户。

输入

用户进入“大额动账”页面后，可按动账时间、资金流向、核心客户号、客户姓名、阅读状态查询大额动账提醒。

处理

（一）操作说明

1. 用户进入“大额动账”页面后，系统默认展示当前登录岗位权限范围内的大额动账提醒列表。

2. 查询条件区的操作按钮有：“重置”“查询”。点击“查询”，系统按当前输入条件返回符合条件的大额动账列表结果；点击“重置”，系统清空当前输入条件并将列表恢复初始状态。

3. 左上角统一操作按钮区的操作按钮有：“批量已读”。点击“批量已读”，系统将所有未读状态的数据更新为已读。

4. 列表行内统一操作按钮区的操作按钮有：“已读”“邀约计划”。

4.1 仅未读状态的数据展示“已读”按钮，点击后系统将当前提醒更新为已读。

4.2 点击“邀约计划”按钮后，系统右侧抽屉式打开新建邀约计划页面。邀约计划功能详见“公共功能-客户联络-邀约计划”章节。

（二）规则与约束

1. 系统通过跑批获取客户每笔大额资金变动数据，资金变动包括转入、转出。

2. 大额动账阈值支持在“经营管理/提醒设置”页面设置，总行统一设置，并支持分支行个性化调整。

活动量指标说明

活动量指标包含电访、面访、企微和短信，具体指标如下：

电访计划目标：

电访计划月目标 = ∑(客户经理名下管户的或机构辖下的各等级管户数 × 该等级月最低接触次数)。

电访计划季目标 = ∑(客户经理名下管户的或机构辖下的各等级管户数 × 该等级月最低接触次数) * 3

电访计划年目标 = ∑(客户经理名下管户的或机构辖下的各等级管户数 × 该等级月最低接触次数) * 12

电访实际完成：

客户经理的实际完成统计是基于当月/当季/当年客户经理的实际有效电话数。

机构的实际完成统计是基于辖下客户经理当月/当季/当年的实际有效电话数。

面访计划目标：

面访计划月目标 = ∑(客户经理名下管户的或机构辖下的各等级管户数 × 该等级月最低接触次数)。

面访计划季目标 = ∑(客户经理名下管户的或机构辖下的各等级管户数 × 该等级月最低接触次数) * 3

面访计划年目标 = ∑(客户经理名下管户的或机构辖下的各等级管户数 × 该等级月最低接触次数) * 12

面访实际完成：

客户经理的实际完成统计是基于当月/当季/当年客户经理的实际有效面访数。

机构的实际完成统计是基于辖下客户经理当月/当季/当年的实际有效面访数。

企微计划目标：

企微计划月目标 = ∑(客户经理名下管户的或机构辖下的各等级管户数 × 该等级月最低接触次数)。

企微计划季目标 = ∑(客户经理名下管户的或机构辖下的各等级管户数 × 该等级月最低接触次数) * 3

企微计划年目标 = ∑(客户经理名下管户的或机构辖下的各等级管户数 × 该等级月最低接触次数) * 12

企微实际完成：

客户经理的实际完成统计是基于当月/当季/当年客户经理的实际有效企微数。

机构的实际完成统计是基于辖下客户经理当月/当季/当年的实际有效企微数。

短信目标计划：

短信计划月目标 = ∑(客户经理名下管户的或机构辖下的各等级管户数 × 该等级月最低接触次数)。

短信计划季目标 = ∑(客户经理名下管户的或机构辖下的各等级管户数 × 该等级月最低接触次数) * 3

短信计划年目标 = ∑(客户经理名下管户的或机构辖下的各等级管户数 × 该等级月最低接触次数) * 12

短信实际完成；

机构的实际完成统计是基于辖下客户经理当月/当季/当年的实际有效短信数。
