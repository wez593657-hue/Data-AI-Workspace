# 流失挽回规则记忆卡片

## 一、流失等级定义

| 流失等级 | 规则 | 判定基准 |
|----------|------|----------|
| **轻度流失** | 上月金融资产月日均达标，上月末金融资产余额不达标 | 上月末 |
| **重度流失** | 上上月金融资产月日均达标（上月金融资产月日均不达标），上月末金融资产余额不达标 | 上上月末 |

## 二、数据来源

| 数据项 | 来源表 | 字段 | 说明 |
|--------|--------|------|------|
| 上月月日均 | `DWS_CUST_ASSE_LIAB` | `AUM_BAL`（`BAL_TYPE='2'`） | 上月末日期 |
| 上上月月日均 | `DWS_CUST_ASSE_LIAB` | `AUM_BAL`（`BAL_TYPE='2'`） | 上上月末日期 |
| 上月末余额 | `DWS_CUST_ASSE_LIAB` | `AUM_BAL`（`BAL_TYPE='1'`） | 上月末日期 |
| T-1日余额 | `DWS_CUST_ASSE_LIAB` | `AUM_BAL`（`BAL_TYPE='1'`） | V_SYSDAT-1 |
| 当前客户等级 | `DWS_CUST_LVL_INFO` | `CUST_LVL` | 当前日期 |
| 客户等级达标阈值 | 规则配置 | - | 优质/财富1/财富2: 5万；贵宾: 30万；私行: 50万/100万/300万 |

## 三、接触状态

| 接触状态 | 规则 |
|----------|------|
| 已接触 | 存在营销记录（`ADS_MKT_REC_INFO`） |
| 未接触 | 无营销记录 |

## 四、挽回状态

| 挽回状态 | 规则 |
|----------|------|
| 已挽回 | T-1日金融资产余额达标 |
| 未挽回 | T-1日金融资产余额不达标 |

## 五、统计指标

| 指标 | 说明 | 计算方式 |
|------|------|----------|
| 客户数 | 各流失等级下的客户数量 | COUNT |
| 已接触客户 | 各流失等级下已接触的客户数量 | COUNT(CNTCT_STATE='1') |
| 接触率 | 已接触客户数 / 客户数 × 100% | COUNT(CNTCT_STATE='1') / COUNT × 100 |
| 已挽回客户 | 各流失等级下已挽回的客户数量 | COUNT(RESCUE_STATE='1') |
| 挽回率 | 已挽回客户数 / 客户数 × 100% | COUNT(RESCUE_STATE='1') / COUNT × 100 |
| 已挽回金融资产 | T-1日金融资产余额达标的客户，从月初~T-1日金融资产新增总金额 | SUM(RESCUED_FINA_ASSET) |

## 六、已挽回金融资产计算

**口径**：T-1日金融资产余额达标的客户，从月初~T-1日金融资产新增总金额

**单个客户计算**：`RESCUED_FINA_ASSET = MAX(T-1日余额 - 上月末余额, 0)`

**明细表字段**：`RESCUED_FINA_ASSET`（已挽回金融资产金额）

**统计表汇总**：`SUM(RESCUED_FINA_ASSET)`

## 七、统计维度

| 维度 | 说明 |
|------|------|
| 统计周期 | 月(M)、季(Q)、年(N) |
| 统计对象 | 机构（向上汇总）、客户经理 |
| 流失等级 | 轻度流失(1)、重度流失(2) |
| 个人/对公 | 个人、对公 |

## 八、客户等级范围

优质、财富1、财富2、贵宾、私行
