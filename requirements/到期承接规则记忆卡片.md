## 二、功能概述

### 功能描述

支持客户经理和管理人员查看到期承接分析结果及客户明细，帮助用户识别产品到期客户、掌握承接与转化情况并支撑后续经营跟进。

### 统计周期

| 周期类型 | 说明 | 选择范围 |
|----------|------|----------|
| 月(M) | 默认当月 | 1月～当月 |
| 季(Q) | 默认当季 | 一季度～当季 |
| 年(N) | 默认当年 | 当年～当年（历史数据保留3年） |

### 产品范围

- **存款**：定期存款（核心系统的定期存款与智能存款的定期存款）
- **理财**：理财产品

## 四、到期窗口计算规则

### 规则说明

```
统计周期 = 客户{存款｜理财}到期后30天（含）

如果同客户该月有多笔{存款｜理财}到期：
  - 取下一笔到期日减1（如果30天内有下一笔到期）
  - 否则取最后一笔到期日后30天（含）
```

### 到期窗口计算逻辑

```sql
TAKE_END_DT_30D = 
  NVL(
    (SELECT MIN(n.EXPR_DT) - 1
      FROM TMP_CDR_DTL_MATURE_SRC n
      WHERE n.CUST_ID = g.CUST_ID
        AND n.STATIS_TYP = g.STATIS_TYP
        AND n.EXPR_DT > g.LAST_EXPR_DT
        AND n.EXPR_DT <= g.LAST_EXPR_DT + 30),
    g.LAST_EXPR_DT + 30
  )
```

## 五、【全部】页签指标

### 指标列表

| 指标 | 说明 | 计算公式 |
|------|------|----------|
| 已到期客户数 | 本月1号开始，截至T-1日有{存款｜理财}到期的客户数 | COUNT |
| 总到期客户数 | 当期到期的客户总数 | COUNT |
| 已到期金额 | 本月1号开始，截至T-1日有{存款｜理财}金额的汇总 | SUM |
| 总到期金额 | 当月到期客户{存款｜理财}金额的汇总 | SUM |
| 客户承接率 | 已到期客户中，到期资金购买长期化产品比重≥80%的客户占比 | 符合条件客户数 / 已到期客户数 × 100% |
| 资产留存率 | 当前AUM与本月第一笔到期前一日AUM的比率 | 当前AUM / 到期前一日AUM × 100% |
| 资产承接率 | 到期后购买长期化产品金额占到期金额的比例 | Σ(购买金额) / 已到期金额 × 100% |
| 保险转化率 | 到期客户购买保险的金额占比 | 购买保险金额 / 已到期金额 × 100% |
| 理财转存款转化率 | 到期理财客户购买长期化存款的金额占比 | 购买存款金额 / 已到期理财金额 × 100% |

### 客户承接率规则

| 规则项 | 说明 |
|--------|------|
| 统计周期 | 到期后30天（含），按到期窗口规则计算 |
| 长期化产品 | AUM产品分类，剔除开放式理财、活期存款、通知存款和保险 |
| 比重要求 | 购买长期化产品资金 / 本月到期资金 ≥ 80% |
| 购买日期 | 按产品起息日 |

### 资产承接率规则

| 规则项 | 说明 |
|--------|------|
| 统计周期 | 到期后30天（含），按到期窗口规则计算 |
| 长期化产品 | AUM产品分类，剔除活期化理财、活期存款、通知存款和保险 |
| 计算方式 | Σ(单客户购买金额) / 本月已到期总额 |

### 理财转存款转化率规则

| 规则项 | 说明 |
|--------|------|
| 统计周期 | 到期后30天（含），按最后一笔到期日+30天 |
| 长期化存款 | 剔除活期存款、通知存款的其他产品 |

## 八、DATA_DATE说明

### 语义变更

**变更前**：周期未结束时=快照日期（V_SYSDAT），周期结束时=周期结束日期

**变更后**：始终=周期结束日期（M-月末，Q-季末，N-年末）

### 周期结束日期计算

| 周期类型 | 计算方式 |
|----------|----------|
| 月(M) | LAST_DAY(当前日期) |
| 季(Q) | ADD_MONTHS(TRUNC(当前日期, 'Q'), 3) - 1 |
| 年(N) | ADD_MONTHS(TRUNC(当前日期, 'YYYY'), 12) - 1 |
