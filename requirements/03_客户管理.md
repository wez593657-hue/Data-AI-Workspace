# 03 客户管理

## 客户管理


我的客户

我的客户-列表

功能描述

支持理财经理或客户经理查询和检索名下管户客户清单，并提供单客快捷操作及批量移交能力，助力客户经理高效掌握客户情况并开展日常跟进。

输入

输入操作按钮包括“查询”、“重置”、“高级查询”，用于提交查询条件、清空当前条件以及展开更多筛选项。

处理

（一）操作说明

1. 用户进入“我的客户”列表页面后，系统默认加载本人名下客户清单。

2. 页面默认展示“客户列表”tab内容，用户点击“客户盘点”tab后，可切换查看客户盘点页面内容，详见“我的客户-客户盘点”章节。

3. 默认展示“客户列表”查询条件：用户可按核心客户号、客户姓名、联系电话、是否关注等条件输入或选择查询项，其中“是否关注”为下拉选择，枚举值为已关注、未关注。

4. 用户可在查询条件区操作

点击“查询”，系统按当前输入条件返回符合条件的客户列表结果。

点击“重置”，系统清空当前输入条件并恢复初始状态。

点击“高级查询”，跳转至高级查询功能，详见：高级查询与策略章节。

5. 用户可在列表左上角统一操作按钮区操作

点击“客户移交”，从右侧抽屉式弹出客户移交界面，详见“我的客户-客户移交”章节。

6. 用户可在列表行内统一操作按钮区操作

点击“客户邀约”，从右侧抽屉式弹出客户邀约界面，详见“我的客户-邀约计划”章节。

点击“加提醒”，从右侧抽屉式弹出加提醒界面，详见“我的客户-加提醒”章节。

点击“加关注”，从右侧抽屉式弹出加关注界面，详见“我的客户-加关注”章节。

（二）规则与约束

1. 客户经理原则上仅查询本人名下客户清单。客户如果是白名单客户，则客户名称、资产、负债数据全部脱敏，如姓名“陈*奎”（陈*），手机号、证件号按照统一的脱敏规则脱敏，资产余额、负债余额、账户余额、日均全部脱敏，如：****.**。（白名单客户需要在后台手工初始化，后续新增的白名单通过科技的运维单调整）。

2. 支持按核心客户号、客户姓名、联系电话、是否关注等条件进行查询筛选，其中“是否关注”为下拉筛选项，枚举值为已关注、未关注。

3. 当用户未勾选客户直接发起批量类操作时，系统应弹出二次确认框，提示文案为：“您当前未勾选客户，系统将对当前筛选出的 {X} 位客户进行批量【客户移交】操作。此操作不可逆，请确认是否继续？”。

4. 当用户勾选部分或全部客户执行批量类操作时，系统应仅对已勾选客户执行处理，并弹出二次确认框，提示文案为：“您将对已勾选的 {X} 位客户进行批量【客户移交】操作。请确认是否继续？”

5. 查询展示需兼顾业务便捷性与数据安全，手机号、证件号等敏感字段展示时需脱敏。

6. 列表行内操作栏提供“客户邀约”、“加提醒”、“加关注”功能，列表左上角统一操作按钮区提供“客户移交”等批量或全局操作。

7. 页面右上角提供“查询”、“重置”、“高级查询”等通用查询操作，左上角提供“客户移交”、“列设置”等统一操作按钮。

8.列表默认按客户等级降序排序；支持活期余额、定期余额、理财余额、贷款余额升序/降序排序切换。

（三）权限说明

1. 理财经理、客户经理仅查询和检索本人名下管户客户清单。

输出

我的客户列表默认展示核心客户号、客户姓名、联系电话、活期余额、定期余额、贷款余额字段，用户可以通过列设置，个性化定义展示列与列顺序。

我的客户-客户盘点

功能描述

支持理财经理或客户经理对名下管户客户的资产负债、产品持有、客户生命周期分布等进行统计分析与客户盘点，助力客户经理快速识别客户经营机会和客户变化情况。

输入

不涉及

处理

（一）操作说明

1.用户进入“我的客户”列表页面后，系统默认加载本人名下客户清单。页面默认展示“客户列表”tab内容，用户点击“客户盘点”tab后，可切换查看客户盘点页面内容。

2、展示指标包括：客户数、AUM 余额、活期余额、定期余额、理财余额及贷款余额等盘点指标。

3、展示图表包括：客户分层分析、AUM构成分析、存款期限分析、产品持有分析、贷款构成分析、客户生命周期分析。

（二）规则与约束

1. 我的客户盘点默认展示当前理财经理或客户经理名下管户客户的数据。

2. 客户盘点结果以统计分析为主。

3. 客户盘点按照T+1实现。

4. 保险余额口径：首期保费加缴费期间内续期保费累计为保险金额，缴费期满，保费金额置为0。（若在缴费期间的宽限期后还未有续费记录的，宽限期后将保费金额置为0）。

（三）权限说明

1. 理财经理或客户经理仅可查看本人名下管户客户的盘点数据。

我的客户-客户移交

功能描述

支持理财经理和客户经理主动发起名下客户移交并跟踪移交流程处理结果，提供客户归属调整能力，保障客户资源有序流转和持续经营。

输入

处理

（一）操作说明

1. 用户在“我的客户”列表页点击“客户移交”后，系统从右侧抽屉式弹出客户移交界面。

2. 用户选择移交至：本机构或客户经理，并填写移交原因后提交申请。

3. 系统校验必输项、移交范围和移交目标信息后发起客户移交流程并提示，提示文案：“您已成功发起客户移交流程，请在工作台/我的申请模块，查看流程审批进度！”。

4. 流程提交后，系统返回我的客户列表页面。

（二）规则与约束

1. 客户移交支持移交至本机构转为未分配状态，或移交给其他客户经理。当选择移交客户经理时，显示本机构下辖的客户经理（理财经理）人员，展示格式：“工号-姓名”。

2. 移交申请需按组织架构流转审批，未经审批通过不得直接变更客户归属。

本网点移交流程：理财经理发起 – 网点负责人审批。

3. 用户发起的客户移交流程，审批通过或不通过，流程都会结束，若审批不通过，想要移交同样的客户，需再次发起新的客户移交流程。

4. 客户移交不允许跨机构移交。

5. 当用户未选择客户直接发起客户移交时，系统应默认按当前筛选结果范围处理，并给予明确确认提示（提示文案详见“我的客户-列表”章节）。

6. 移交后需向接收方发送提醒。

7. 客户移交审批通过后，应记录客户归属变更历史，同步清理移交客户在原客户经理客群中的关联关系。

（三）权限说明

1. 客户经理仅可对本人名下客户发起移交申请。

机构未分配客户

机构未分配客户-列表

功能描述

支持理财经理或客户经理查询机构辖下五级以下（上月月日均10万以下）未分配客户并发起申领操作，提升营销人员的营销积极性与归属处理效率。

输入

支持通过核心客户号、客户名称、联系电话、客户等级筛选未分配客户，其中核心客户号和联系电话支持精确查询。

处理

（一）操作说明

1. 用户进入机构未分配客户列表后，系统默认展示机构辖下五级以下（上月月日均10万以下）未分配客户。

2. 用户可在查询条件区操作

用户输入查询条件后点击“查询”，系统按条件检索未分配客户。

用户点击“重置”后，系统清空当前筛选条件并恢复默认值。

3. 用户可在列表左上角统一操作按钮区操作

用户在列表中选择客户后，点击“申领”按钮，可进入客户认领（申领）流程，从右侧抽屉式弹出申领界面，详见“机构未分配客户-申领”章节。

（二）规则与约束

1.理财经理或客户经理只能申领本机构辖下的未分配客户，不能跨机构申领客户。

2. 信贷客户（有贷户）的管户客户经理自动同步信贷系统，信贷客户不在CRM系统申领管户关系，如需变更信贷客户的管户关系，需先在信贷系统中变更管户关系，CRM会自动同步。

3. 查询展示需兼顾业务便捷性与数据安全，手机号、证件号等敏感字段展示时需脱敏。

4.页面右上角提供“查询”、“重置”等通用查询操作，左上角提供“申领”统一操作按钮。

5.列表默认按客户等级降序排序；支持客户等级升序/降序排序切换

（三）权限说明

1.理财经理或客户经理仅可申领机构辖下的未分配客户。

【定期存款】页签

定期存款到期金额：客户当期已到期定期存款金额。

定期存款到期总金额：客户当期到期定期存款总金额。

定期存款承接率：当期已到期的定期存款金额，有多少到期资金在规定时间内购买了定期存款（不含通知存款）的产品。客户承接率=符合上述规则/已到期总金额*100%。

理财产品到期：推荐三个匹配的理财产品。若都不匹配则不推荐。
