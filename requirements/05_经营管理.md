# 05 经营管理

## 经营管理


重点经营视图

重点经营视图-到期承接

功能描述

支持客户经理和管理人员查看到期承接分析结果及客户明细，帮助用户识别产品到期客户、掌握承接与转化情况并支撑后续经营跟进。

输入

用户进入“重点经营视图”后，系统默认展示“到期承接”tab内容。

到期承接客户明细列表，用户可按“归属机构”、“客户名称”、“客户等级”、“接触状态”、“承接状态”等条件筛选客户明细。

处理

（一）操作说明

1. 用户进入“重点经营视图”后，系统默认展示“到期承接”tab，并按月加载统计结果。

点击“潜力提升”tab，详见重点经营视图-潜力提升章节。

点击“流失挽回”tab，详见重点经营视图-流失挽回章节。

2. 系统展示存款、理财产品的到期承接结果，包括客户数（未到期/已到期）、到期金额（未到期/已到期）、客户承接率、资产留存率、资产承接率、理财转化率、保险转化率指标。

3. 用户可在查询条件区操作

用户可按“归属机构”、“客户名称”、“客户等级”、“接触状态”、“承接状态”等条件筛选客户明细。

点击“查询”，系统返回符合条件的结果。

用户点击“重置”后，系统清空当前筛选条件并恢复默认值。

4. 系统展示客户明细列表，点击“客户名称”可跳转至对应客户视图。

（二）规则与约束

1. 到期承接分析当前按当月、当季、当年三个时间切片展示，当选择当月时，下拉框支持选择月份，默认当月，选择区间1月～当月；当选择季度时，下拉框支持选择季度，默认为当季，选择区间一季度～当季；当选择当年时，默认为当年（如2026年），选择区间2026年～2026年（以后2027年……都支持选择，历史数据保留3年）。

2. 本期到期承接的产品范围覆盖存款（定期存款：核心系统的定期存款与智能存款的定期存款）和理财。

3. 展示要求：已到期客户数与总到期客户数合在一起展示，例如：“25/50（展示已到期客户数占比）”；已到期金额与总到期金额的展示上同。客户承接率、资产留存率等指标分开独立展示，例如：“客户承接率 25.12% （展示较上月）”。

4. 指标口径说明：以下指标根据用户不同汇总的数据范围也不同，例如客户经理只汇总名下管户客户的到期客户数或金额。机构管理用户需按照客户的归属机构统计辖下到期客户数或金额（也是管户口径）。指标分为“全部”、“定期存款”两个页签、支持按照tab页签。

【全部】页签

已到期客户数：当期到期的客户即本月1号开始，截至T-1日有{存款｜理财}到期的客户数。

总到期客户数：当期到期的客户总数。

已到期金额：当期已到期客户即本月1号开始，截至T-1日有{存款｜理财}金额的汇总。

总到期金额：当月到期客户{存款｜理财}金额的汇总。

客户承接率：当月已到期的{存款｜理财}客户中，有多少人的到期资金在规定时间内购买了长期化的产品金额且比重大于等于80%。客户承接率=符合上述资格客户数/已到期客户数*100%。

1)	统计周期=客户{存款｜理财}到期后30天（含），如果同客户该月有多笔{存款｜理财}到期，按最后一笔到期日后30天（含）。

2)	长期化的产品指AUM 产品分类中产品，剔除开放式理财、活期存款、通知存款、保险；

3)	比重大于等于80%以上的客户=统计周期客户购买长期化产品的资金/客户本月到期理财资金≥80%；

4)	购买日期明确：{存款｜理财}按产品起息日

资产留存率：当月已到期的{存款｜理财}客户，当前AUM/本月第一笔到期前一日AUM*100%，该留存率可以超过100%。

资产承接率：当月已到期的客户中，到期后的客户在规定期限内购买的长期化产品金额，占{存款｜理财}到期金额的比例。资产承接率=符合上述条件的资金/已到期{存款｜理财}的金额*100%，该承接率允许超过100%。

1)	统计周期=客户{存款｜理财}到期后14天（含），如果同客户该月有多笔理财到期，按最后一笔到期日后14天（含）

2)	长期化的产品指AUM 产品分类中产品，剔除活期化理财、活期存款、通知存款、保险；

3)	资产承接率=Σ(单客户统计周期购买产品的资金)/本月已到期{存款｜理财}总额；

保险转化率：当月到期的{存款｜理财}客户，在统计周期内购买保险的金额百分比，保险转化率=符合条件的资金/已到期{存款｜理财}金额*100%。

1)	统计周期=客户{存款｜理财}到期后30天（含），如果同客户该月有多笔{存款｜理财}到期，按最后一笔到期日后30天（含）；

理财转存款转化率：当月到期的理财客户，在统计周期内购买长期化存款的金额百分比，存款转化率=符合条件的资金/已到期理财金额*100%

1)	统计周期=客户理财到期后30天（含），如果同客户该月有多笔理财到期，按最后一笔到期日后30天（含）；

2)	长期化存款指剔除活期存款、通知存款的其他产品

【定期存款】页签

定期存款到期金额：当期已到期定期存款金额。

定期存款到期总金额：当期到期定期存款总金额。

定期存款承接率：当期已到期的定期存款金额，有多少到期资金在规定时间内购买了定期存款（不含通知存款）的产品。定期存款承接率=符合上述规则/已到期总金额*100%。

1)	统计周期=客户定期存款到期后30天（含），如果同客户该月有多笔定期存款到期，按最后一笔到期日后30天（含）。

2)	购买日期明确：定期存款按产品起息日。

存款转理财转化率：当月到期的存款客户，在统计周期内购买长期化理财的金额百分比，理财转化率=符合条件的资金/已到期存款金额*100%

1)	统计周期=客户存款到期后30天（含），如果同客户该月有多笔存款到期，按最后一笔到期日后30天（含）；

2)	长期化理财剔除开放式理财

3. 默认按客户等级降序排序，涉及金额字段和日期字段应支持升序/降序切换。

（三）权限说明

1. 管理用户与客户经理都有权查看，管理用户查看机构辖下产品到期经营数据，客户经理查看本人名下管户产品到期经营数据。

到期承接统计指标

理财到期承接统计指标

重点经营视图-潜力提升

功能描述

支持客户经理和管理岗查看临界客户的潜力提升分析结果及客户明细，帮助用户识别可升级客户并支撑后续分层提升经营。

输入

用户进入“重点经营视图”后，点击“潜力提升”tab，系统加载潜力提升分析数据。

潜力提升客户明细列表，用户可按“归属机构”、“客户名称”、“临界等级”、“接触状态”、“达标状态”等条件筛选客户明细

处理

（一）操作说明

1. 用户进入“重点经营视图”后，点击“潜力提升”tab，系统展示潜力提升分析数据。

2. 用户可切换不同临界等级查看总客户数、已接触客户（接触率）、月均达标（达标率）、时点达标（达标率）分析指标。

3. 用户可在查询条件区操作

用户可查看潜力客户明细，并根据临界等级、接触状态、达标状态等条件筛选数据。

点击“查询”，系统返回符合条件的结果。

用户点击“重置”后，系统清空当前筛选条件并恢复默认值。

4. 系统展示客户明细列表，点击“客户名称”可跳转至对应客户视图

（二）规则与约束

1. 潜力提升分析当前按当月、当季、当年三个时间切片展示，当选择当月时，下拉框支持选择月份，默认当月，选择区间1月～当月；当选择季度时，下拉框支持选择季度，默认为当季，选择区间一季度～当季；当选择当年时，默认为当年（如2026年），选择区间2026年～2026年（以后2027年……都支持选择，历史数据保留3年）

2. 潜力/临界提升，面向“待提升到下一等级”的临界客户开展分析。

2. 当前临界提升分析以临界等级为核心维度，至少支持优质、财富1、财富2、贵宾、私行四类临界客户。

3. 指标包括总总客户数、已接触客户（接触率）、月均达标（达标率）、时点达标（达标率）。

4. 指标口径说明：以下指标根据用户不同汇总的数据范围也不同，例如客户经理只汇总名下管户客户潜力提升分析。机构用户需按照客户的归属机构统计辖下客户的潜力提升分析（也是管户口径）

总客户数：按上月金融资产月日均等级判断各层级客户数，[4.5，5万）为临界优质，[27，30万）为临界财富1，[45，50万）为临界财富2，[90，100万）为临界贵宾，[270，300万）为临界私行。

月均达标：当月金融资产月日均达标优质/财富1/财富2/贵宾/私行客户数。

月均达标率：各层级临界客户本月月日均资产达标客户数/对各应临界的总客户数*100%。

时点达标：T-1日金融资产余额分别达标潜力/优质/财富/贵宾/私行客户数。

时点达标率：各层级T-1日金融资产余额分别达标潜力/优质/财富/贵宾/私行客户数/对应各临界的总客户数*100%。

已接触客户：当月各层级临界客户，有过面访、电话、短信、企微的客户数。

接触率：已接触客户／当月对应临界的总客户数*100%。

5. 默认按客户等级降序排序，涉及金额字段和日期字段应支持升序/降序切换

（三）权限说明

1. 管理用户与客户经理都有权查看，管理用户查看机构辖下潜力提升经营数据，客户经理查看本人名下管户潜力提升经营数据。

重点经营视图-流失挽回

功能描述

支持客户经理和管理岗查看流失挽回分析结果及客户明细，帮助用户识别流失风险客户并支撑后续挽回经营。

输入

用户进入“重点经营视图”后，点击“流失挽回”tab，系统加载流失挽回分析数据。

流失挽回客户明细列表，用户可按“归属机构”、“客户名称”、“客户等级”、“接触状态”、“挽回状态”等条件筛选客户明细。

处理

（一）操作说明

1. 用户进入“重点经营视图”后，点击“流失挽回”tab，系统展示流失挽回分析结果。

2. 系统按月统计流失挽回数据，并区分 轻度流失等级、流失等级，默认展示流失，点击流失进行页面的切换。

3. 用户可查看不同等级的在流失、流失流失等级下客户数、已接触客户（接触率）、已挽回客户（挽回率）、已挽回金融资产余额。

4. 用户可在查询条件区操作

用户可按“归属机构”、“客户名称”、“客户等级”、“接触状态”、“挽回状态”等条件筛选客户明细。

点击“查询”，系统返回符合条件的结果。

用户点击“重置”后，系统清空当前筛选条件并恢复默认值。

5. 系统展示客户明细列表，点击“客户名称”可跳转至对应客户视图。

（二）规则与约束

1. 流失挽回分析当前按当月、当季、当年三个时间切片展示，当选择当月时，下拉框支持选择月份，默认当月，选择区间1月～当月；当选择季度时，下拉框支持选择季度，默认为当季，选择区间一季度～当季；当选择当年时，默认为当年（如2026年），选择区间2026年～2026年（以后2027年……都支持选择，历史数据保留3年）。

2. 轻度流失规则为：上月金融资产月日均达标，上月末金融资产余额不达标。

3. 重度流失规则为：上上月金融资产月日均达标（上月金融资产月日均不达标），上月末金融资产余额不达标。

客户等级：优质，财富1，财富2，贵宾，私行

4. 按等级统计，分析在轻度流失、重度流失流失等级下客户数、已接触客户（接触率）、已挽回客户（挽回率）、已挽回金融资产余额指标。

5. 指标口径说明：以下指标根据用户不同汇总的数据范围也不同，例如客户经理只汇总名下管户客户流失分析。机构用户需按照客户的归属机构统计辖下客户的流失分析（也是管户口径）。

客户数：当前客户等级、当前流失等级（轻度流失等级、轻度流失）的客户数。

已接触客户：当月当前客户等级、当前流失等级的客户，有过面访、电话、短信、企微的客户数。

接触率：当前客户等级、当前流失等级已接触客户/当前客户等级、当前流失等级（流失、流失）客户数*100%

已挽回客户：T-1日金融资产余额达标客户数。

挽回率：当前客户等级、当前流失等级已挽回客户/当前客户等级、当前流失等级（流失、流失）客户数*100%

已挽回金融资产：T-1日金融资产余额达标的客户，从T0～T-1日金融资产新增总金额。

6. 默认按客户等级降序排序，涉及金额字段和日期字段应支持升序/降序切换

（三）权限说明

1. 管理用户与客户经理都有权查看，管理用户查看机构辖下流失挽回经营数据，客户经理查看本人名下管户流失挽回经营数据。

重点经营视图-新客经营

功能描述

支持用户查看新客经营概览及客户明细，帮助用户掌握新客户接触和 KYC 完成情况，并支撑新客经营。

输入

用户进入“重点经营视图”后，点击“新客经营”tab，系统加载新客经营分析结果。

用户可按“归属机构”、“客户名称”、“客户等级”、“接触状态”、“KYC状态”等条件筛选新客明细。

处理

（一）操作说明

1. 用户进入“重点经营视图”后，点击“新客经营”tab，系统展示新客经营分析结果。

2. 系统按月统计新客经营数据，按0～30天、30～100天、100～180天、全部四类，分类展示。Tab默认为30天及以内，支持切换30~100天、100~180天、全部。

3. 用户可查看不同新客周期（0～30天、30～100天、100～180天、全部）下金融资产区间的客户数、已接触客户（接触率）、KYC客户（KYC客户完成率）。

4. 用户可在查询条件区操作

用户可按“归属机构”、“客户名称”、“客户等级”、“接触状态”、“ KYC状态”等条件筛选客户明细。

点击“查询”，系统返回符合条件的结果。

用户点击“重置”后，系统清空当前筛选条件并恢复默认值。

5. 系统展示客户明细列表，点击“客户名称”可跳转至对应客户视图。

6. 系统展示新客经营核心指标，包括“已接触客户”和“KYC客户”。

（二）规则与约束

1. 新客经营分析当前按当月、当季、当年三个时间切片展示，当选择当月时，下拉框支持选择月份，默认当月，选择区间1月～当月；当选择季度时，下拉框支持选择季度，默认为当季，选择区间一季度～当季；当选择当年时，默认为当年（如2026年），选择区间2026年～2026年（以后2027年……都支持选择，历史数据保留3年）

2. 新客经营围绕“新客户经营”场景展开，重点关注新客首轮接触和 KYC 完善情况。

3. 新客经营的核心指标包括“本月已接触”和“KYC客户”。

4. 指标口径说明：统计不同新客周期（0～30天、30～100天、100～180天、全部）内的客户。

新客户：以核心客户号开立至180天为行内新客户。

0~5万元客户数：当前时点金融资产余额层级，T-1日资产余额在[0,5万）

5~30万元客户数：当前时点金融资产余额层级，T-1日资产余额在[5,30万）

支持用户创建新的目标任务，录入任务基础信息、考核对象、考核指标及目标值，形成后续审批和执行的基础数据。

处理

（一）操作说明

1. 用户在“目标任务管理”列表点击“新增”按钮右侧抽屉式打开“新增目标任务”页面。页面顶部展示“新增目标任务”步骤，一共四个步骤，使用四个图标代表四步，按顺序展示，图标初始状态为灰色，用户操作到某一步，则当前步骤以及前序步骤的图标切换为主题色，标识用户当前所在的步骤。顶部下面区域展示用户需要输入的信息。

2. 第一步“设置基础信息”。录入任务名称、任务持续时间（任务开始日期-任务结束日期）、任务类型、任务说明等基础信息。页面右下角按钮区，点击“取消”按钮关闭“新增目标任务”页面，点击“保存并下一步”按钮保存用户输入的数据并进入第二步。

3. 第二步“选择考核指标”。考核指标列表回显所属部门、对应考核对象类型下的全部指标，指标可多选。页面右下角按钮区，点击“上一步”按钮返回到第一步，点击“保存并下一步”按钮保存用户输入的数据进入第三步“选择考核对象”。

4. 第三步“选择考核对象”。用户可按机构树选择辖下机构或客户经理。可多选。可向辖下跨机构层级选择考核对象，例如总行选择某网点作为考核对象。页面右下角按钮区，点击“上一步”按钮返回到第二步，点击“保存并下一步”按钮保存用户输入的数据进入第四步“指标分解明细”。

5. 第四步“指标分解明细”。页面以表格的形式展示考核对象、考核指标。“列”表头为“考核指标”，反显第二步中选择的指标。“行”表头为“考核对象”，反显第三步中选择的考核对象。考核对象与考核指标交叉的表格，为输入框，必输，格式为整数，输入后的数值即作为某考核对象的某考核指标的指标值。最后一行特殊处理，将各考核对象的指标值相加。页面右下角按钮区，点击“上一步”按钮返回到第三步，点击“完成”按钮校验必填项及数据格式，如满足要求则生成目标任务并toast提示“已生成目标任务”，如不满足要求则弹窗提示“请检查数据，指标值均为必填，且需输入整数”，弹窗仅展示“确定”按钮，点击“确定”按钮关闭弹窗。

6. 用户返回上一步时，页面需回显已保存的数据。

7. 目标任务创建成功后系统生成任务记录，初始状态为“待提交”。

（二）规则与约束

1. 新增页面全部字段均为必输项，缺失时不可保存，不可进入下一步。

2. 创建成功后任务尚未生效，需提交审核并审批通过后，系统自动下发。

3. 同一机构、同一任务周期类型（年度/季度/月度）、同一开始日期/结束日期下，不可重复下发目标任务。

4. 第二步“选择考核指标”页面需按指标分类展示指标。

（三）权限说明

1. 仅总行业务岗、分行业务岗、支行业务岗和网点负责人可新建目标任务。

2. 用户仅可为其本部、辖内机构或客户经理创建任务。

分解

功能描述

支持机构将上级下发的目标任务分解至辖下机构或客户经理，实现目标任务逐级传导和责任落实。

输入

“选择考核对象”：

“指标分解明细”：

处理

（一）操作说明

1. 用户在目标任务管理列表行内点击“分解”按钮，右侧抽屉式打开“任务分解”页面。

2. “任务分解”页面与“新建目标任务”的第三步“选择考核对象”、第四步“指标分解明细”页面要素相同，详见“目标任务管理-新增”章节。

3. 在第三步“选择考核对象”，用户根据机构树选择辖下机构或客户经理作为分解对象，可跨机构层级选择分解对象。

4. 在第四步“指标分解明细”，考核对象回显第三步用户选择的分解对象，考核指标回显上级下发任务时的考核指标。

5. 系统校验通过后保存分解结果，并形成新的下级任务。

（二）规则与约束

1. 仅上级下发的“待分解”、“已分解待提交”状态的任务支持分解。

2. 分解任务时，可向辖下跨机构层级选择考核对象，例如分行选择辖内某网点的客户经理作为考核对象。

3. 分解指标值可在上级指标值基础上加码，分解后的指标值相加不能小于上级下发的指标值。

4. 任务分解后，状态为“已分解待提交”，此状态支持重新分解；提交审核后不支持重新分解；审核通过后自动下发到分解对象（考核对象）。

（三）权限说明

1. 仅总行业务岗、分行业务岗、支行业务岗和网点负责人可执行分解。

2. 用户仅可对本机构接收的上级任务进行分解，不得跨机构分解。

提交审核

功能描述

支持用户将新建、编辑或分解后的目标任务提交审批流程，经审批通过后正式生效。

输入

处理

（一）操作说明

1. 用户在目标任务管理列表选中“待提交”、“已分解待提交”状态的任务后点击“提交审核”按钮。

2. 系统校验任务完整性后，发起审批流程并将状态更新为“审核中”。

3. 审批完成后，系统根据结果更新任务状态。

（二）规则与约束

1. 仅“待提交”、“已分解待提交”状态任务可提交审批。

2. 目标任务均为同级审批。如果是本机构创建的任务，则由本机构内非创建人外的管理岗审批；如果是本机构分解的上级任务，则由本机构内非分解人外的管理岗审批。

3. 审批被拒绝后，任务状态更新为“审批未通过”，允许编辑后再次提交。

4. 审批通过后，系统自动下发任务，任务方可生效并进入实际执行状态判断。

（三）权限说明

1. 仅总行业务岗、分行业务岗、支行业务岗和网点负责人可提交审核。
