智能搜索

功能描述

支持用户通过统一智能搜索入口快速检索权限范围内的客户信息，并支持跳转客户360视图，帮助用户提升客户查找访问效率。

输入

用户进入智能搜索入口后，系统默认进入客户搜索模式，并展示历史搜索记录。

用户输入姓名、手机号或证件号等关键词后，系统返回符合条件的客户搜索结果。

处理

（一）操作说明

1. 用户进入智能搜索入口后，系统默认展示客户搜索界面。

2. 若存在历史搜索记录，系统在当前页面展示最近搜索记录列表。

3. 用户输入客户姓名、手机号码或证件号码后，点击“选择”或回车键，系统按关键词返回匹配的客户结果。点击“^”或“ˇ”切换按钮，向上或向下选择记录，点击“Esc”关闭按钮，关闭客户搜索界面。

4. 用户点击某条客户搜索结果的客户名称，系统跳转至对应客户360视图。

5. 用户点击某条历史记录后，系统按该记录内容重新执行搜索。

6. 用户点击删除角标后，系统直接删除对应历史搜索记录。

（二）规则与约束

1. 智能搜索所有用户均可使用。

2. 本期实现客户搜索，客户搜索支持模糊查询。

3. 搜索结果仅返回当前用户权限范围内可查看的客户数据。

4. 历史搜索记录最多保留最近 10 条（在输入文本框下每个搜索关键词一个小胶囊，共展示10个胶囊），若出现重复搜索项，仅保留最近一次搜索内容。

5. 删除历史记录时无需二次确认，点击胶囊右侧删除角标，后直接删除。

6. 搜索结果中的手机号、证件号等敏感信息需按系统统一脱敏规则展示。

（三）权限说明

1. 用户仅可搜索并查看本人权限范围内的客户。

2. 智能搜索不突破原有客户数据权限边界。

输出

系统输出客户搜索界面、历史搜索记录

系统输出客户搜索结果及对应跳转结果。

机构树

功能描述

支持 PC 端各类页面统一使用机构树组件进行机构层级展示和机构选择，帮助用户在权限范围内快速定位目标机构并完成查询、分配或统计分析操作。

输入

业务页面加载机构选择能力时，系统根据当前用户的数据权限初始化机构树。

用户可通过展开节点、输入机构名称关键词或直接勾选节点完成机构选择。

处理

（一）操作说明

1. 业务页面加载机构树组件时，系统按当前用户岗位的数据权限（归属机构）初始化机构树根节点。

2. 用户可点击父级节点展开或收起下级机构。

3. 用户输入机构关键词后，系统在权限范围内按机构名称进行模糊匹配，并定位或过滤符合条件的机构节点。

4. 当页面为单选模式时，用户一次仅可选择一个机构节点。

5. 当页面为多选模式时，用户可同时选择多个机构节点，供后续查询、统计或目标下发使用。

6. 用户确认选择后，系统将所选机构结果返回业务页面，由页面继续执行查询、保存或分配动作。

（二）规则与约束

1. 机构树根节点应根据当前用户的数据权限初始化，不得展示超出权限范围的机构。

2. 管理用户通常可查看其归属机构及下级机构；普通营销用户通常仅可查看本人权限范围内机构节点。

3. 机构树需支持展开/收起层级浏览。

4. 机构树需支持机构名称模糊查询。

5. 机构树选择模式应支持由接入页面配置为单选或多选。

6. 在查询类场景下，接入页面可根据业务规则限制仅允许选择一个机构；在目标下发、统计分析等场景下，可支持多选。

7. 多选模式下，是否允许同时勾选父级与子级机构，由接入页面业务规则决定；组件本身需预留兼容能力。

8. 机构树返回结果应同时包含机构名称和机构唯一标识，便于业务页面后续处理。

（三）权限说明

1. 机构树组件本身不单独定义业务权限，机构可见范围完全继承当前用户数据权限。

2. 用户仅可选择其权限范围内可见机构，不得通过搜索或展开方式获取越权机构。

输出

系统输出用户权限范围内的机构树结构及当前选中的机构结果。

分页组件

功能描述

支持 PC 端各类列表页面统一使用分页组件进行结果浏览和跨页切换，帮助用户在大数据量场景下快速查看目标记录并保持交互体验一致。

输入

当页面查询结果超过单页展示上限时，系统在列表底部加载分页组件。

用户可通过页码、上一页、下一页及每页条数等分页控件切换列表展示结果。

处理

（一）操作说明

1. 当列表查询结果超过单页展示上限时，系统在列表底部展示分页组件。

2. 系统默认加载第一页数据，并同步展示当前页、总页数和总记录数等分页信息。

3. 用户点击“上一页”时，系统加载当前页的前一页数据。

4. 用户点击“下一页”时，系统加载当前页的后一页数据。

5. 用户点击指定页码时，系统直接切换到对应页并刷新列表结果。

6. 用户切换“每页条数”后，系统按新的单页条数重新计算总页数，并从第一页重新加载数据。

（二）规则与约束

1. 分页组件适用于结果集较大的列表场景；当数据量不足单页时，页面可不展示或弱化分页组件。

2. 分页应作用于完整查询结果集，不得仅对当前页局部排序后再分页。

3. 当列表已应用排序条件时，应先按全量结果完成排序，再执行分页切片。

4. 每页条数应支持统一配置，每页条枚举值：10条、20条、30条、50条、100条。

5. 当用户切换查询条件、排序条件或筛选条件后，分页应重置到第一页。

6. 当前页已为第一页时，“上一页”不可继续向前翻页；当前页已为最后一页时，“下一页”不可继续向后翻页。

7. 分页切换仅刷新列表结果区域，不应导致整页重复初始化或清空已生效的查询条件。

8. 当输入前往X页，大于实际分页时，应跳转到分页的最后一页。

（三）权限说明

1. 分页组件本身不单独控制数据权限，数据权限沿用接入页面的业务权限规则。

2. 分页组件仅对当前用户有权查看的结果集执行分页，不得突破原有权限边界

输出

系统输出分页状态信息并展示对应页的列表结果。

列设置

功能描述

支持用户自定义我的客户/辖内客户等客户列表的展示列和列顺序，并保存个人展示偏好，助力用户按经营需要高效查看客户信息。

输入

处理

（一）操作说明

1. 用户在列表页右上角，点击“列设置”按钮后，下拉悬浮展示列设置界面。

2. 用户可勾选或取消勾选可展示字段，并调整已选字段的展示顺序，列表的展示栏位即时调整。

（二）规则与约束

1. 列设置中的列表栏位选项来源于标签宽表中的标签：核心客户号、客户名称、年龄、手机号码、客户等级、AUM余额、活期存款余额、定期存款余额、理财产品余额、贷款余额、社保卡、川工之家。

2. 列设置应仅影响当前用户个人的列表展示效果，不影响其他用户（缓存在本地浏览器，不保存到数据库，重装浏览器/换其他浏览器/清除浏览器缓存后，需要重新设置列）。

3. 列设置支持展示列、拖动列进行列排序，若列表展示的列太多，支持横向滚动，且支持锁定操作列。

4. 已保存的列设置在用户下次进入列表时应自动生效。

（三）权限说明

1. 用户可维护本人使用范围内的客户列表列设置

输出

列设置支持勾选可展示字段、调整列顺序，并通过确定或取消按钮完成本次配置。

发送短信

功能描述

支持用户对本人权限范围内的单个客户或多个客户发送短信，支持选择可用短信模板、自动带出模板内容，帮助客户经理提升客户触达效率并满足合规管控要求。

输入

用户在XX列表页左上角统一操作按钮区点击“发送短信”后，系统从右侧抽屉式弹出发送短信页面。

用户选择选择一个或多个客户，点击“发送短信”后，系统从右侧抽屉式弹出发送短信页面。选择“模板类型”、“短信模板”，系统自动反显模板内容。

处理

（一）操作说明

1. 用户在XXX列表页勾选一个或多个客户后，点击左上角统一操作按钮区的“发送短信”，系统从右侧抽屉式弹出发送短信页面。

2. 用户未勾选客户直接点击“发送短信”时，系统默认本次发送范围为当前筛选条件下的全部客户，并弹出二次确认提示。

3. 用户选择“模板类型”、“短信模板”后，系统自动反显模板内容，并按客户信息、客户经理信息、机构信息完成模板变量替换预览。

4. 用户点击“确定”后，系统校验模板可用性、发送次数限制及必输项完整性。校验通过后，调用短信平台的短信发送接口并返回发送结果。

5. 用户点击“取消”后，系统关闭当前发送短信页面，不保留本次未提交内容。

（二）规则与约束

1. 发送短信支持对单个客户或多个客户发起，发送对象仅限当前用户权限范围内客户。

2. 当用户未勾选客户直接发起发送短信时，系统应弹出二次确认框，提示文案为：“您当前未勾选客户，系统将对当前筛选出的 {X} 位客户进行批量【发送短信】。此操作不可逆，请确认是否继续？”

3. 当用户勾选部分或全部客户发起发送短信时，系统应弹出二次确认框，提示文案为：“您将对已勾选的 {X} 位客户进行批量【发送短信】。请确认是否继续？”

4. 短信模板必须来源于短信模板管理，仅允许选择审批通过的模板。

5. 发送次数限制：本系统全局控制，在7天内同一个客户只能发送一条短信。

6. 模板内容为模板管理模块统一维护结果，发送页面仅反显内容，不支持客户经理在本页面自由修改模板正文。

7. 短信发送应通过日志留痕，便于后续查询和审计。

，

1. 客户经理、理财经理仅可对本人权限范围内客户发起发送短信申请。

2. 用户仅可查看和选择权限范围内可使用的短信模板。

打电话

功能描述

支持用户在各业务场景中，通过移动端（企微端）一键拨号发起电话联络，帮助用户提升客户触达效率并沉淀基础联络留痕。

输入

略

处理

（一）操作说明

1、用户在移动端（企微端）工作日历、客户视图等业务场景中，通过点击“一键拨号”按钮，给客户拨打电话。

2、挂断电话之后，返回移动端原页面默认打开营销反馈的录入界面。详见：公共功能/客户联络/营销反馈章节

（二）规则与约束

1、拨打电话需要进行二次确认，提示文案为：“您确认要给客户拨打电话吗？”

2、一键拨号功能调用手机本身的拨号能力，系统在本项目仅记录拨打电话的时间（应保存电话拨打操作记录，至少包括客户经理工号、客户姓名、拨号时间），不记录通话时长和通话录音。给客户拨打的电话需整合到客户视图的联络信息。

3、打电话功能一次仅支持针对一位客户发起，不支持批量电话外呼。

4、打电话功能仅可针对当前用户权限范围内客户发起，不得越权拨打非本人管户客户电话。

5、联系电话属于敏感信息，页面展示时需脱敏，支持通过小眼睛查看完整的电话号码，但系统应该记录用户本次的查看操作记录。

（三）权限说明

1. 客户经理、理财经理仅可对本人权限范围内管户客户发起电话联系。

2. 客户经理、理财经理仅可查看和拨打本人有权查看的客户联系方式，不得突破原有客户数据权限边界。
