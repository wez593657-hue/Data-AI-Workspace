# 06 产品管理

## 产品管理


产品管理-列表

功能描述

支持总行业务岗统一管理零售银行部（零售条线）产品库，整合核心、信贷、理财、保险等产品数据，并通过目录体系检索产品、产品上下架控制和提供产品视图，为全行产品营销提供统一产品底座。

输入

“产品管理列表”页面左侧区域为“产品目录”，右侧区域为“产品列表”。

左侧“产品目录”输入项：

右侧“产品列表”输入项：

处理

（一）操作说明

1. 用户进入“产品管理-产品管理”页面后，页面左侧为“产品目录”区域，右侧为“产品列表”区域。

2. 用户在左侧产品目录区域可输入关键词搜索目录，点击搜索框内的“搜索”图标执行搜索，若用户未输入关键词则toast提示“请输入目录名称”；点击搜索框内的“重置”图标，清空关键词并重置目录列表。

2.1. 产品目录按层级展示，点击产品目录名称左侧的“展开”按钮，展开该目录的下级目录，并将按钮切换为“收起”按钮；点击“收起”按钮，收起该目录的下级目录，并将按钮切换为“展开”按钮。所有目录都默认展开，按钮默认为“收起”。

2.2 点击产品目录名称，右侧“产品列表”刷新为当前目录下的产品数据。

3. 右侧产品列表查询条件区的查询条件有：产品名称、产品编号、上下架。

4. 右侧产品列表查询条件区的操作按钮有：查询、重置。

4.1 点击“查询”，系统按当前输入条件返回符合条件的产品列表结果。

4.2 点击“重置”，系统清空当前输入条件并恢复初始状态。

5. 点击产品名称或产品编号后，系统打开“产品视图”页面查看产品详情。

6. 右侧产品列表行内统一操作按钮区的操作按钮有：上架、下架

6.1 若该行产品为上架状态，则上架按钮置灰不可点击，若该行产品为下架状态，则下架按钮置灰不可点击。

6.2 点击上架按钮， 页面中间弹窗提示“是否确认将该产品上架？”，用户点击“确认”按钮后执行上架操作。

6.3 点击下架按钮， 页面中间弹窗提示“产品下架后将不会在移动端展示，是否确认下架？”，用户点击“确认”按钮后执行下架操作。

（二）规则与约束

1. 产品目录体系当前按以下目录分类。

2. 产品基础数据统一从上游系统获取，按T+1频率同步至产品管理模块。产品基础数据详见“产品视图-产品简介”章节。

3. 产品目录名称按“目录名称（n）”格式展示，其中n为当前目录包含的产品数量。

4. 产品下架后，不在移动端展示，“产品管理”模块仍可查看和编辑维护。

5. 列表字段有：产品名称、产品编号、产品类型、产品来源。

（三）权限说明

1. 菜单权限限定为总行业务岗。

2. 数据权限覆盖全行零售产品数据，包括核心、信贷、理财、保险系统产品产品。

3. 非授权用户不可进入产品管理模块或执行上下架维护。

产品上架/下架

功能描述

支持用户对产品执行上架和下架控制，统一管理产品移动端场景中的可见状态。

输入

处理

（一）操作说明

1. 用户在产品列表中定位目标产品后，根据当前状态点击“上架”或“下架”按钮。

1.1 若该行产品为上架状态，则上架按钮置灰不可点击，若该行产品为下架状态，则下架按钮置灰不可点击。

1.2 点击上架按钮， 页面中间弹窗提示“是否确认将该产品上架？”，用户点击“确认”按钮后执行上架操作。

1.3 点击下架按钮， 页面中间弹窗提示“产品下架后将不会在移动端展示，是否确认下架？”，用户点击“确认”按钮后执行下架操作。

1.4 上架、下架操作完成后页面需toast提示“上架成功”、“下架成功”，上架、下架操作失败后需toast提示“操作失败”。

2. 系统校验产品当前状态与操作类型是否匹配，校验通过后更新产品上架状态。

3. 状态更新成功后，系统刷新当前行展示结果。

（二）规则与约束

1. 上架、下架操作均基于单条产品执行，不支持当前清单范围内的批量上下架。

2. 已上架产品不可重复上架，已下架产品不可重复下架。

3. 下架后的产品在PC端“产品管理”模块页面可见，但在移动端不可见。

4. 上下架状态变更需记录操作人、操作时间和操作结果。

（三）权限说明

1. 仅总行业务岗可执行产品上架/下架。

产品介绍

功能描述

产品介绍tab页分三个区域展示，分别为产品简介、产品特点、适用人群。支持用户维护产品简介及优势、产品特点、适用人群等营销展示内容，为前台产品详情页提供统一的产品介绍信息。

输入

处理

（一）操作说明

1. 用户在“产品视图”页面切换至“产品介绍”tab页后，系统展示产品简介、产品特点、适用人群三个区域。

2. 用户点击任一区域标题右侧“编辑”按钮后，进入编辑状态并使用富文本编辑器维护内容，并隐藏“编辑”按钮，新增展示“保存”、“取消”按钮。

3. 用户点击任一区域标题右侧“保存”按钮后，系统校验输入的内容是否超过最高字符限制，如果超过则toast提示“最多不超过1000个字符，请重新输入”，如果没超过则保存对应区域内容，隐藏“保存”、“取消”按钮，展示“编辑”按钮，并返回查看状态。

4. 用户点击任一区域标题右侧“取消”按钮后，不执行编辑操作，隐藏“保存”、“取消”按钮，展示“编辑”按钮，并返回查看状态。

（二）规则与约束

1. 产品介绍内容允许按区域分别维护，互不覆盖。

2. 富文本内容应支持基础排版能力，并在保存后按统一展示规范反显。

3. 若某区域未维护内容，则在移动端展示时可不展示对应模块，“产品视图”页面展示空模块，可编辑。

（三）权限说明

1. 仅总行业务岗可编辑产品介绍内容。

营销设置

功能描述

支持用户维护产品营销话术、封面、海报和营销手册等营销素材，为产品营销和前台展示提供统一内容支撑。

输入

营销设置tab页面的输入项：

“营销手册上传”弹窗的输入项：

处理

（一）操作说明

1. 用户在“产品视图”页面切换至“营销设置”tab页后，系统展示产品营销话术、产品封面、产品海报、产品营销手册四个区域。

2. 用户点击营销话术区域“编辑”按钮后，进入编辑状态，可通过富文本编辑器维护话术内容，并隐藏“编辑”按钮，新增展示“保存”、“取消”按钮。

2.1点击“保存”按钮后，系统校验输入的内容是否超过最高字符限制，如果超过则toast提示“最多不超过1000个字符，请重新输入”，如果没超过则保存对应区域内容，隐藏“保存”、“取消”按钮，展示“编辑”按钮，并返回查看状态。

2.2 点击“取消”按钮后，不执行编辑操作，隐藏“保存”、“取消”按钮，展示“编辑”按钮，并返回查看状态。

3. 用户点击“上传图片”后打开系统自带的“选择文件”弹窗，可上传产品封面、PC端海报或移动端海报，系统按对应区域规则校验文件格式、大小和数量，不满足要求则toast提示“请上传不超过10M的jpg/png图片”。

4. 用户点击营销手册区域“上传”按钮后，系统以右侧抽屉形式打开“营销手册上传”页面，用户选择手册类型并上传文件，系统需校验文件格式，如果不满足要求则toast提示“请上传docx/excel/pdf/pptx格式的文件”。页面右下角展示“确定”、“取消”按钮。

4.1 点击“确定”按钮，上传文件到营销手册的列表区域，并关闭“营销手册上传”页面。

4.2 点击“取消”按钮，不执行上传文件操作，直接关闭“营销手册上传”页面。

5. 上传成功后，营销手册按类型回显在列表中，并显示文档名称和上传时间。

（二）规则与约束

1. 产品封面仅允许上传1张图片，再次上传时覆盖原图片。

2. 产品封面建议分辨率为380*240px；PC端海报建议分辨率1920*1080px；移动端海报建议分辨率320*497px。

3. 每类海报最多上传10张；超出数量限制时系统应阻止继续上传并toast提示“最多支持上传10张海报”。

4. 营销手册文件格式仅支持docx、excel、pdf、pptx。

5. 营销手册列表按“全部、Word、ppt、Excel、pdf”五个tab页展示，默认显示“全部”。

（三）权限说明

1. 仅总行业务岗可编辑营销设置。
