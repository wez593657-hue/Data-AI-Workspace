## 二、功能概述

### 功能描述

支持客户经理和管理岗查看临界客户的潜力提升分析结果及客户明细，帮助用户识别可升级客户并支撑后续分层提升经营。

### 统计周期

| 周期类型 | 说明 | 选择范围 |
|----------|------|----------|
| 月(M) | 默认当月 | 1月～当月 |
| 季(Q) | 默认当季 | 一季度～当季 |
| 年(N) | 默认当年 | 当年～当年（历史数据保留3年） |

### DATA_DATE说明

**语义**：统计周期最大日期 = 最新跑批日期V_SYSDAT（用户已确认）

## 四、临界等级定义

### 等级划分规则

| 临界等级码 | 临界等级名称 | 上月月日均AUM范围 | 达标阈值 |
|------------|--------------|-------------------|----------|
| '03' | 临界优质 | [4.5万, 5万) | ≥5万 |
| '04' | 临界财富1 | [27万, 30万) | ≥30万 |
| '05' | 临界财富2 | [45万, 50万) | ≥50万 |
| '06' | 临界贵宾 | [90万, 100万) | ≥100万 |
| '07' | 临界私行 | [270万, 300万) | ≥300万 |

### 临界等级判断逻辑

```sql
CASE
  WHEN p.AUM_BAL >= 45000 AND p.AUM_BAL < 50000 THEN '03'
  WHEN p.AUM_BAL >= 270000 AND p.AUM_BAL < 300000 THEN '04'
  WHEN p.AUM_BAL >= 450000 AND p.AUM_BAL < 500000 THEN '05'
  WHEN p.AUM_BAL >= 900000 AND p.AUM_BAL < 1000000 THEN '06'
  WHEN p.AUM_BAL >= 2700000 AND p.AUM_BAL < 3000000 THEN '07'
END AS LVL_CRIT
```

## 五、指标列表

### 明细表指标

| 指标 | 说明 | 计算方式 |
|------|------|----------|
| 临界等级 | 按上月月日均AUM判断的临界等级 | LVL_CRIT字段 |
| 接触状态 | 当月有过有效接触（面访、电话、短信、企微） | CNTCT_STATE='1' |
| 时点达标状态 | T-1日时点AUM达到对应临界等级阈值 | QUAL_STATE='1' |

### 统计表指标

| 指标 | 说明 | 计算公式 |
|------|------|----------|
| 总客户数 | 各临界等级下的客户总数 | COUNT(*) |
| 月均达标客户数 | 当前月日均AUM达标的客户数 | COUNT(MTH_AVG_QUAL_STATE='1') |
| 月均达标率 | 月均达标客户数 / 总客户数 × 100% | SUM(MTH_AVG_QUAL_STATE='1') / COUNT(*) × 100 |
| 时点达标客户数 | T-1日时点AUM达标的客户数 | COUNT(PNT_QUAL_STATE='1') |
| 时点达标率 | 时点达标客户数 / 总客户数 × 100% | SUM(PNT_QUAL_STATE='1') / COUNT(*) × 100 |
| 已接触客户数 | 当月有过有效接触的客户数 | COUNT(CNTCT_STATE='1') |
| 接触率 | 已接触客户数 / 总客户数 × 100% | SUM(CNTCT_STATE='1') / COUNT(*) × 100 |

### 达标判断逻辑

**月均达标**（统计过程计算）：
```sql
CASE
  WHEN (LVL_CRIT = '03' AND NVL(M.AUM_BAL, 0) >= 50000)
    OR (LVL_CRIT = '04' AND NVL(M.AUM_BAL, 0) >= 300000)
    OR (LVL_CRIT = '05' AND NVL(M.AUM_BAL, 0) >= 500000)
    OR (LVL_CRIT = '06' AND NVL(M.AUM_BAL, 0) >= 1000000)
    OR (LVL_CRIT = '07' AND NVL(M.AUM_BAL, 0) >= 3000000)
  THEN '1'
  ELSE '0'
END
```

**时点达标**（明细过程计算）：
```sql
CASE
  WHEN (LVL_CRIT = '03' AND PNT_AUM_BAL >= 50000)
    OR (LVL_CRIT = '04' AND PNT_AUM_BAL >= 300000)
    OR (LVL_CRIT = '05' AND PNT_AUM_BAL >= 500000)
    OR (LVL_CRIT = '06' AND PNT_AUM_BAL >= 1000000)
    OR (LVL_CRIT = '07' AND PNT_AUM_BAL >= 3000000)
  THEN '1'
  ELSE '0'
END
```

## 八、三年历史清理规则

### 清理条件

```sql
-- 明细表
DELETE FROM ADS_CUST_POTN_UPGRADE_CUST_DTL D
 WHERE D.DATA_DATE < TO_CHAR(ADD_MONTHS(TRUNC(TO_DATE(V_SYSDAT, 'YYYYMMDD'), 'YYYY'), -36), 'YYYYMMDD');

-- 统计表
DELETE FROM ADS_CUST_POTN_UPGRADE_STATIS T
 WHERE T.DATA_DATE < TO_CHAR(ADD_MONTHS(TRUNC(TO_DATE(V_SYSDAT, 'YYYYMMDD'), 'YYYY'), -36), 'YYYYMMDD');
```

### 规范说明

- 使用字符串比较，避免对目标表DATA_DATE做函数转换导致索引失效（需求文档第47行要求）
- 历史期末快照最多保留三年
