智能搜索

功能描述

支持用户通过统一智能搜索入口快速检索权限范围内的客户信息，并支持跳转客户360视图，帮助用户提升客户查找访问效率。

输入

用户进入智能搜索入口后，系统默认进入客户搜索模式，并展示历史搜索记录。

用户输入姓名、手机号或证件号等关键词后，系统返回符合条件的客户搜索结果。

处理

（一）操作说明

1. 用户进入智能搜索入口后，系统默认展示客户搜索界面。

2. 若存在历史搜索记录，系统在当前页面展示最近搜索记录列表。

3. 用户输入客户姓名、手机号码或证件号码后，点击“搜索”，系统按关键词返回匹配的客户结果。

4. 用户点击某条客户搜索结果的客户名称，系统跳转至对应客户360视图。

5. 用户点击某条历史记录后，系统按该记录内容重新执行搜索。

6. 用户点击删除角标后，系统直接删除对应历史搜索记录。

我的申请

我的申请-列表

功能描述

支持用户集中查看本人发起的各类申请记录，按审批状态分类跟踪申请结果和流程进度，提升申请查询与跟进效率。

输入

“审批中”“审批通过”“审批未通过”三个Tab页面，点击“筛选”按钮，系统打开筛选面板，三个Tab页面查询条件一致，即申请类型、流程状态、申请时间。

处理

（一）操作说明

1. 用户登录移动端后，在首页点击“我的申请”入口，进入“我的申请”列表页面，页面按“申请状态”分为“审批中”“审批通过”“审批未通过”三个Tab页面。系统默认展示“审批中”Tab列表。

2. 点击“筛选”按钮，系统打开筛选面板，筛选面板的操作按钮有：“查询”“重置”。

2.1 点击“查询”，系统按当前输入条件返回符合条件的申请列表结果。

2.2 点击“重置”，系统清空当前输入条件并恢复初始状态。

2.3 “审批中”“审批通过”“审批未通过”列表页的数据分开查询。

4. 用户点击“我的申请”列表中的申请卡片，系统进入“申请详情”页面。申请详情详见“我的申请-详情”章节。

（二）规则与约束

1. “审批中”列表展示申请提交后审批流程尚未结束的数据；“审批通过”列表展示审批流程已结束且全部节点均为“同意”的数据；“审批未通过”列表展示审批流程中任一审批节点为“未通过”的数据。

2. 申请类型有：“客户移交”“客户申领”“目标任务”“沙龙活动”“短信模板”。

3. “流程状态”取值为“正常”“异常”。系统处理审批流程中的数据错误并标记为异常的，流程状态显示“异常”，其他情况均显示“正常”。

4. 列表默认按“申请时间”倒序排序。

（三）权限说明

1. 菜单权限为业务岗、网点负责人、客户经理岗可见。

2. 数据权限为：申请者仅可查看本人提交的申请数据；审批者（即在审批节点上的用户）可查看所有申请者提交的申请数据。

输出

“审批中”tab页：

“审批通过”tab页：

“审批未通过”tab页：

功能描述

支持用户查看单笔申请的详情，即审批信息和审批流程，准确掌握申请内容（即审批信息）、审批节点状态及处理进度。

处理

（一）操作说明

1. 用户在“我的申请”列表点击“申请卡片”，系统进入“申请详情”页面。

2. 页面上部分展示“审批信息”，下部分展示“审批流程”。

2.1 “审批信息”：不同的申请类型，展示的审批信息不同，详见此模块的“输出”章节。

2.2 “审批流程”：按顺序展示申请人、审批节点。申请人展示申请人、申请时间。审批节点按系统配置的审批顺序展示审批节点，未完成审批的节点只展示审批人，且节点图标为灰色，已完成审批的节点展示审批人、审批时间、审批意见，且节点图标为亮色。

3. 用户点击“返回”按钮后，系统关闭“申请详情”页面返回上一页。

（二）规则与约束

1. 页面上部分“审批信息”根据申请类型展示不同明细内容；下部分“审批流程”按审批顺序展示申请人节点及审批节点信息。

2. 审批流程中，申请人节点展示申请人、申请时间；审批节点按系统配置顺序展示。未完成审批的节点仅展示审批人字段，节点图标为灰色；已完成审批的节点展示审批人、审批时间、审批意见，节点图标为亮色。

3. 沙龙活动参与人中的手机号码属于敏感信息，需脱敏展示，可按“保留前3位和后4位，中间4位以*替代，如138****1234”的规则显示。

（三）权限说明

1. 申请者可查看本人提交申请的详情。

2. 位于审批节点上的审批者可查看该申请的详情。

待办审批-列表

功能描述

支持用户集中查看本人待审批及已审批的申请事项，并按审批状态分类执行审批处理或查看审批结果。

输入

“审批中”“已审批”两个Tab页面，点击“筛选”按钮，系统打开筛选面板，两个Tab页的查询条件一致，即申请类型、申请时间、申请人、流程状态进行查询。

处理

（一）操作说明

1. 用户进入“待办审批”列表页面后，页面按“审批状态”分为“审批中”“已审批”两个Tab页面。系统默认展示“审批中”Tab页面。

2. 点击“筛选”按钮，系统打开筛选面板，筛选面板的操作按钮有：“查询”“重置”。点击“查询”，系统按当前输入条件返回符合条件的列表结果；点击“重置”，系统清空当前输入条件并将列表恢复初始状态。“审批中”、“已审批”列表页的数据分开查询。

3. “审批中”列表申请卡片上面的操作按钮有：审批。点击“审批”按钮，系统进入待审批的“审批详情”页面。

4. “已审批”列表申请卡片上面的操作按钮有：查看。点击“查看”按钮，系统进入已审批的“审批详情”页面。

（二）规则与约束

1. “审批中”展示“待当前节点审批”且“用户位于当前审批节点中”的申请数据；“已审批”展示“用户位于审批节点中”且“该审批节点已审批”的申请数据。审批节点中的用户共享审批信息，即节点中的任一用户完成审批后，节点中的其他用户也可查看该节点的“审批意见”等数据。

2. 审批中列表按“申请时间”倒序排序；已审批列表按“审批时间”倒序排序。

3. 流程状态取值为“正常”“异常”。系统处理审批流程中的数据错误并标记为异常的，流程状态显示“异常”，其他情况均显示“正常”。

（三）权限说明

1. 菜单权限为总分支行业务岗、网点负责人。

2. 数据权限为：审批者（即在审批节点上的用户）可查看流转到该节点的所有申请数据。

3. 审批者可查看并处理待办审批列表。

待办审批-详情

功能描述

支持审批者在审批详情中查看申请明细和审批流程，录入审批意见并执行同意或拒绝操作，完成当前审批节点处理。

输入

处理

（一）操作说明

1. 用户在“审批中”列表卡片上点击“审批”按钮，系统进入待审批的“审批详情”页面。

2. 用户在“已审批”列表卡片上点击“查看”按钮，系统进入已审批的“审批详情”页面。

3. “审批详情”页面分为“审批信息”“审批意见”“审批流程”三个区域。待审批的“审批详情”页面，“审批意见”可编辑；已审批的“审批详情”页面，“审批意见”不可编辑，仅做数据回显。

3. 待审批的“审批详情”页面的操作按钮有：“拒绝”“同意”。

3.1 点击“拒绝”按钮，系统先校验是否已输入审批意见，已输入则执行审批拒绝操作，未输入则toast提示“请输入审批意见”。

3.2 点击“同意”按钮，系统先校验是否已输入审批意见，已输入则执行审批同意操作并推动流程进入下一审批节点，未输入则toast提示“请输入审批意见”。

（二）规则与约束

2. “审批详情”页面中部“审批意见”区域，在“待审批”场景为可输入文本框，在“已审批”场景为当前节点审批意见只读回显。

3. “审批详情”页面下部分“审批流程”的展示内容，与“我的申请”详情页下部分的“审批流程”保持一致。详见“移动端（企微）功能需求-首页-待办及消息提醒-我的申请-详情”章节。

（三）权限说明

1. 仅位于当前审批节点上的审批者可在待审批详情页执行“同意”或“拒绝”操作。

2. 已审批详情页仅用于查看，审批意见不可修改，不可重复审批。

输出

大额动账

功能描述

支持用户集中查看管辖客户的大额资金变动提醒，及时识别转入、转出等异常或重点资金流动并跟进客户。

输入

用户进入“大额动账”页面后，点击“筛选”按钮，系统打开筛选面板，可按动账时间、资金流向、核心客户号、客户姓名、阅读状态查询大额动账提醒。

处理

（一）操作说明

销户提醒：客户注销了名下所有I类账户、II类户、III类户的客户，进行提醒。

工作日历

功能描述

支持用户在首页中查看本人权限范围内的当日工作计划，并通过更多入口进入完整日历页，帮助用户快速掌握近期工作计划。

输入

用户进入“首页”页面后，默认展示当天的待办工作计划摘要。

用户可点击“更多”，跳转至完整工作日历视图，可切换日期查看对应日期下的工作计划和完成情况。

处理

（一）操作说明

1、用户进入“首页”页面后，默认展示用户当日待办工作计划摘要。

2、工作日历模块默认展示当日前 3 条待办工作计划。

3、用户点击“邀约计划”，详见公共功能/客户联络/邀约计划章节。

4、用户点击“更多”后，跳转至工作日历更多页面，展示完整的工作日历和工作计划。

用户选择不同日期后，系统刷新并展示对应日期下的工作任务计划和完成情况。

（二）规则与约束

1、首页/工作日历模块，每天默认仅展示当日前 3 条待办工作计划，按照时间的先后顺序排序。

2、若当日无工作计划，系统应展示“当日暂无工作计划”。

3、工作日历中的任务来源包括客户经理自建的邀约计划（面访、电访、短信和企微）、客户提醒、沙龙活动。

4、用户仅能查看自己创建的邀约计划（面访、电访、短信和企微）、客户提醒、沙龙活动。

5、工作日历更多页面分为上下两部分

上曾展示当月完整的工作日历，其中：无工作计划的日期下方无标记。有工作计划的日期下方，工作计划全部完成标记为绿色，如“●”，工作计划未全部完成标记为灰色，如“●”。

下层展示工作计划，支持按照面访、电访、短信和企微切换查询

默认展示当天，分上午、下午分别展示其中的的工作计划及完成情况。

6、各类工作计划的完成条件

面访：需要进行营销反馈，详见：公共功能/客户联络/面访章节。

电访：仅支持移动端拨打电话，打完电话后需要进行营销反馈，详见：公共功能/客户联络/打电话章节。

短信：完成短信发送，详见：公共功能/客户联络/发送短信章节

企微：完成信息补录，详见：公共功能/客户联络/企微章节

提醒：完成已读。

沙龙活动：完成沙龙活动的活动反馈，详见：营销管理/沙龙活动管理/活动反馈章节。

（三）权限说明

1、客户经理仅可查看本人名下工作计划，不得查看超出本人权限范围的工作计划。

2、管理用户仅可查看机构辖下的沙龙活动。

功能区

功能描述

支持用户按个人使用习惯配置并展示权限范围内的常用功能快捷入口，帮助用户快速进入高频业务页面并提升日常操作效率。

输入

用户进入“首页”页面，展示已配置的快捷入口列表。

用户点击快捷入口区域“更多”按钮后，跳转至“快捷入口”设置界面，用户可选择菜单、调整顺序，并点击“确定”保存配置。

快捷入口配置界面

处理

（一）操作说明

1、用户进入“首页”页面后，系统默认展示当前用户已配置的快捷入口菜单。

2、用户点击某个快捷入口菜单后，系统直接进入对应功能页面。

3、用户点击快捷入口区域右侧“更多”按钮后，从右侧抽屉式弹出“快捷入口”配置界面。

4、快捷入口配置界面，在查询条件区下方展示

已选菜单（高亮显示）、支持拖动排序、调整快捷入口的排序顺序，点击右上角“x”角标支持已选菜单的移除。

未选菜单，点击右上角“+”角标支持将未选菜单添加到已选。

5、用户点击“确定”后，系统保存快捷入口配置，关闭侧边栏并返回工作台页面，同时刷新快捷入口区域内容。

6、用户点击“取消”后，系统关闭侧边栏，放弃本次未保存修改并返回工作台页面。

（二）规则与约束

1、快捷入口仅展示当前用户权限范围内可访问的系统功能菜单。

2、快捷入口区域最多可配置 7 个功能菜单+1个“更多”按钮。若已配置数量达到 8 个，用户不可继续添加新的快捷入口，系统应给出数量上限提示。快捷入口默认增加一个功能入口，跳转至客户分级数据报表，查询与下载

3、快捷入口展示顺序以用户最后一次保存的排序结果为准。

4、快捷入口配置应按用户维度单独保存，不同用户之间互不影响。

5、快捷入口配置成功后，应在当前页面立即刷新展示。

6、若某已配置菜单后续被撤销权限，则该菜单不得继续展示在快捷入口区域。

（三）权限说明

1、所有用户均可维护本人菜单权限范围内的菜单，配置快捷入口。不得通过快捷入口突破原有菜单权限边界

输出

系统输出当前用户的快捷入口展示结果及快捷入口配置保存结果，以下是单个菜单组合展示内容。

业绩战报

功能描述

支持用户查看不同统计期间内的重点产品业绩概览，帮助用户快速掌握本人、机构的阶段性经营成果并支撑经营分析。

输入

用户进入“首页/业绩战报”模块，系统加载默认“月度”下的业绩概览结果。

用户可切换“统计期间”查看月度、季度、年度，不同口径下的业绩战报数据。

处理

（一）操作说明

1. 用户进入“首页/业绩战报”页面后，系统默认加载当前统计期间（月度）的业绩概览结果。

2. 用户切换月度、季度、年度后，系统刷新对应统计期间的战报数据。

（二）规则与约束

1. 统计期间支持“月度”、“季度”、“年度”三类口径。

2. 统计范围包括“存款”、“理财”、“保险”、“贷款”四类业绩销量。

3. 机构、客户经理的产品销售数据按管户口径进行汇总统计。

4. 管理用户进入页面时，展示对应的统计期间，“存款”、“理财”、“保险”、“贷款”四类业绩销量，支持下钻，按照总行下钻查看分行、分行下钻查看网点、网点下钻查看客户经理、客户经理下钻查看客户的链路。

5. 客户经理用户进入页面时，展示对应的统计期间，“存款”、“理财”、“保险”、“贷款”四类业绩销量，支持下钻，查看客户的销量。

6. 机构层级、客户经理层级的产品销量，单位为万元，单一客户的产品销量单位为元。

（三）权限说明

1. 客户经理可查看本人名下管户客户的业绩战报数据。

2. 管理用户可查看其权限范围内机构业绩战报数据。
