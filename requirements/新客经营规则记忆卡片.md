## 二、功能概述

### 功能描述

支持用户查看新客经营概览及客户明细，帮助用户掌握新客户接触和KYC完成情况，并支撑新客经营。

### 统计周期

| 周期类型 | 说明 | 选择范围 |
|----------|------|----------|
| 月(M) | 默认当月 | 1月～当月 |
| 季(Q) | 默认当季 | 一季度～当季 |
| 年(N) | 默认当年 | 当年～当年（历史数据保留3年） |

### 新客周期分类

| 周期 | 代码 | 说明 |
|------|------|------|
| 0~30天 | 1 | 开户日距统计日<30天 |
| 30~100天 | 2 | 开户日距统计日≥30且<100天 |
| 100~180天 | 3 | 开户日距统计日≥100且≤180天 |
| 全部 | 4 | 0~180天内新客汇总 |

## 四、新客定义规则

### 定义说明

新客户：以核心客户号开立至180天为行内新客户。

### 开户日期来源

```sql
-- 从客户基本信息表获取核心客户号开立日期
TO_DATE(REPLACE(SUBSTR(c.OPEN_DATE, 1, 10), '-', ''), 'YYYYMMDD')
```

### 新客周期计算（左闭右开）

```sql
CASE
  WHEN 统计日 - 开户日 < 30 THEN '1'   -- 0~30天
  WHEN 统计日 - 开户日 < 100 THEN '2'  -- 30~100天
  WHEN 统计日 - 开户日 <= 180 THEN '3' -- 100~180天
END AS NEW_CUST_CYCLE
```

## 五、统计指标

### 指标列表

| 指标 | 说明 | 计算公式 |
|------|------|----------|
| 新客客户数 | 当前统计周期内的新客总数 | COUNT |
| 已接触客户数 | 当月管户经理有过有效电话、面访、企微和短信接触的客户数 | COUNT(CNTCT_STATE='1') |
| 接触率 | 已接触客户数占总新客数的比例 | 已接触客户数 / 总新客数 × 100% |
| KYC客户数 | KYC完整度≥80%的客户数 | COUNT(KYC_STATE='1') |
| KYC完成率 | KYC客户数占总新客数的比例 | KYC客户数 / 总新客数 × 100% |

### 金融资产区间

| 区间 | 代码 | 说明 |
|------|------|------|
| 0~5万元 | 1 | PNT_AUM_BAL < 50000 |
| 5~30万元 | 2 | 50000 ≤ PNT_AUM_BAL < 300000 |
| 30~100万元 | 3 | 300000 ≤ PNT_AUM_BAL < 1000000 |
| 100~300万元 | 4 | 1000000 ≤ PNT_AUM_BAL < 3000000 |
| 300万元及以上 | 5 | PNT_AUM_BAL ≥ 3000000 |

### 接触状态判断规则

| 规则项 | 说明 |
|--------|------|
| 接触类型 | 电话('1')、面访('2')、企微('3')、短信('4') |
| 接触时间 | 当月（从月初到统计日） |
| 接触人员 | 当前管户客户经理 |

### KYC完整度规则

| 规则项 | 说明 |
|--------|------|
| 统计字段 | 22个KYC字段 |
| 完整度要求 | ≥18个字段不为空（≈81.8%） |
| 录入限制 | 不限制当前管户经理录入 |
