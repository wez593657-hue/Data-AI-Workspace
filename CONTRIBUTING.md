# 贡献指南

## 代码托管

远程仓库地址: `https://github.com/wez593657-hue/Data-AI-Workspace.git`

## 分支策略

| 分支类型 | 命名规范 | 用途 |
|----------|----------|------|
| 主分支 | `master` | 生产环境代码 |
| 开发分支 | `develop` | 开发环境代码 |
| 特性分支 | `feature/{feature-name}` | 新功能开发 |
| 修复分支 | `bugfix/{bug-name}` | Bug修复 |
| 发布分支 | `release/{version}` | 版本发布 |

## 开发流程

```
需求分析 → 设计评审 → 开发实现 → 自动校验 → 代码审查 → 合并发布
```

## 提交规范

### 提交类型

| 类型 | 说明 | 示例 |
|------|------|------|
| `feat` | 新功能 | `feat: 新增客户风险评估表` |
| `fix` | Bug修复 | `fix: 修复DDL字段类型错误` |
| `docs` | 文档更新 | `docs: 更新数据字典规范` |
| `refactor` | 代码重构 | `refactor: 优化校验脚本` |
| `style` | 格式调整 | `style: 统一SQL缩进` |
| `test` | 测试相关 | `test: 添加单元测试` |
| `chore` | 构建/工具 | `chore: 更新依赖` |
| `sync` | 数据同步 | `sync: 同步Mapping文件变更` |

### 提交格式

```
<type>(<scope>): <subject>

<body>

<footer>
```

## 门禁规则

### 第一层：仓库规则

- CODEOWNERS: 指定各模块负责人
- PR模板: 必须填写完整检查清单
- Issue模板: 规范问题描述

### 第二层：标准模板

- 需求文档模板
- 设计文档模板
- SQL模板
- 存储过程模板

### 第三层：检查清单

- 开发检查清单
- 审查检查清单
- 发布检查清单

### 第四层：自动校验

- DDL语法校验
- 数据字典一致性校验
- Mapping完整性校验
- SQL规范校验

### 第五层：人工审批

- 技术负责人审批
- 业务负责人审批
- 安全合规审批

## 开发约束

### AI Agent 使用规则

1. **开发Agent**：负责代码实现，不得执行校验操作
2. **校验Agent**：负责质量检查，不得修改业务代码
3. **审查Agent**：负责代码审查，不得修改业务代码

### 边界约束

- 开发Agent只能在指定目录内操作
- 校验Agent只能读取文件，不得修改
- 审查Agent只能提出建议，不得直接修改

## 代码审查

### 审查流程

1. 提交PR
2. 自动校验通过
3. CODEOWNERS自动分配审查人
4. 审查人审核代码
5. 批准后合并

### 审查标准

- 代码规范性
- 逻辑正确性
- 安全性
- 可维护性

## 版本发布

### 发布流程

1. 创建release分支
2. 执行发布检查清单
3. 测试验证
4. 合并到master
5. 创建标签

## 沟通渠道

- 需求讨论: Issue
- 代码审查: PR评论
- 紧急问题: 直接联系负责人
